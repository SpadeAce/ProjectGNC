# 기획서 전체 UE5 기준 전환

> **날짜**: 2026-03-26
> **작업 유형**: 문서 수정 (기획서 용어/아키텍처 전환)

---

## 작업 요약

`WorkReports/Design/` 하위 기획서 11개를 Unity(C#) 기준에서 UE5(C++) 기준으로 전면 수정했다.
파일 경로, 클래스명, 아키텍처 패턴, API, 코드 예시를 모두 현재 포팅된 UE5 프로젝트 구조에 맞게 변환했다.

| Unity | UE5 |
|-------|-----|
| MonoSingleton\<T\> | UGameInstanceSubsystem (7개) / UWorldSubsystem (3개) |
| ScriptableObject | UDataAsset |
| Protobuf (.bytes) | DataTable (FTableRowBase) |
| MonoBehaviour | AActor / UObject / UUserWidget |
| DontDestroyOnLoad | GameInstanceSubsystem (자동 유지) |

| Unity | UE5 |

| SetActive(false) | SetVisibility / SetActorHiddenInGame / Destroy |
| Coroutine | FTimerHandle / Latent Action |
| event Action\<T\> | Dynamic Multicast Delegate |

## 비고

- 기획서 내 게임 설계(규칙, 수치, 흐름)는 변경하지 않음 — 기술 용어와 아키텍처 매핑만 전환
- 코드 예시는 C# 문법에서 C++ 문법으로 변환
- Inspector → Details 패널, Canvas → UMG Overlay 등 에디터 용어도 함께 전환
- 로드맵의 저장/불러오기 항목을 PlayerPrefs → USaveGame으로 변경
