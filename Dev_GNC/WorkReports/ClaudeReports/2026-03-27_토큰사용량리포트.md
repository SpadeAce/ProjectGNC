## 7. 토큰 소비 패턴 요약

### 많이 소비하는 요인

1. **긴 세션 유지**: 대화가 길어질수록 Cache Read가 누적 증가 (매 API 호출마다 전체 컨텍스트 재전송)
2. **서브에이전트 과다 사용**: 총 토큰의 60%가 서브에이전트. 각 서브에이전트가 독립 컨텍스트를 생성하므로 시스템 프롬프트 + CLAUDE.md 등이 매번 중복
3. **대규모 코드 생성 세션**: Phase별 구현에서 Write(151) + Edit(114)로 대량 코드 생성
4. **컨텍스트 압축 이전의 긴 대화**: autocompact 임계치(167K) 도달 전까지 토큰이 선형 증가

### 절약 팁

| 방법 | 예상 절감 |
|------|----------|
| 세션을 작업 단위로 분리 (1 Phase = 1 세션) | Cache Read 30-50% 감소 |
| Agent 대신 직접 Grep/Glob 사용 (단순 검색 시) | 서브에이전트 토큰 절감 |
| `/compact` 명령으로 수동 컨텍스트 압축 | 누적 Cache Read 감소 |
| 코드 생성 시 파일 단위 확인 줄이기 | Read 호출 감소 |
