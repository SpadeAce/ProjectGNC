# 카메라 이동 범위 제한 기능 구현

- **일시**: 2026-04-07
- **분류**: 기능 추가 (카메라)
- **범위**: GridSubsystem.h/.cpp, CardGameCameraPawn.h/.cpp

---

## 배경

카메라가 타일맵 밖으로 무제한 이동 가능한 상태였다. 맵 크기에 따라 카메라 이동 가능 범위를 자동으로 제한하는 기능이 필요했다.

## 변경 사항

### 1. GridSubsystem — `GetMapBounds()` 추가

- 맵의 월드 좌표 최소/최대값을 반환하는 유틸리티 함수 추가
- `GetMapCenter()`와 동일한 패턴으로 전체 타일의 Min/Max 그리드 좌표를 순회
- 타일 반개 크기(`TileSize * 0.5f`)만큼 패딩하여 타일 가장자리까지 포함

### 2. CardGameCameraPawn — 바운드 클램프

- `bClampToMapBounds` (기본 `true`): 맵 바운드 클램프 활성화 여부
- `CameraBoundsPadding` (기본 `200.0f`): 맵 가장자리로부터의 여유 공간 (월드 유닛)
- `HandlePan()`에서 `AddActorWorldOffset()` 후 위치를 맵 바운드 내로 클램프
- 타일맵이 없는 경우(타일 수 0) 클램프를 건너뛰어 안전하게 처리

## 설계 결정

- **월드 좌표 기준 클램프**: 카메라 Yaw(45도)는 보는 방향만 회전시키고, Pivot 위치는 항상 월드 좌표이므로 Yaw와 무관하게 정확히 동작
- **매 프레임이 아닌 입력 시 클램프**: Tick이 아닌 `HandlePan()` 내에서만 클램프하여 불필요한 연산 방지
- **EditCondition 메타**: `bClampToMapBounds`가 false일 때 `CameraBoundsPadding`이 에디터에서 비활성화됨
