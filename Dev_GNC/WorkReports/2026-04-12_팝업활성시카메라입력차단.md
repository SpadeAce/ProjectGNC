# 팝업 활성 시 카메라 입력 차단

- **일시**: 2026-04-12
- **분류**: 기능 개선 (카메라 / UI 시스템)
- **범위**: CardGameCameraPawn.h, CardGameHUD.h, CardGameHUD.cpp

---

## 배경

PausePopup, BattleResultPopup 등 팝업이 화면을 가리는 상황에서도 카메라 Pan(우클릭 드래그), Rotate(Q/E), Zoom(스크롤) 입력이 그대로 동작하는 문제가 있었다. 팝업이 열려 있는 동안에는 카메라 입력을 완전히 차단해야 한다.

## 원인

- `FInputModeGameAndUI` 모드 사용으로 팝업 표시 중에도 게임 입력이 활성 상태
- 카메라 입력 핸들러에 팝업 상태를 확인하는 로직 없음
- Enhanced Input의 MappingContext가 항상 활성화되어 있음

## 해결 방법

UE5 Enhanced Input의 표준 패턴인 **MappingContext 제거/복원** 방식을 적용했다.

### 접근 방식

HUD의 `OpenPopup`/`ClosePopup`/`CloseAllPopups` 시점에 CameraPawn의 `CameraMappingContext`를 Enhanced Input Subsystem에서 제거하거나 재등록한다.

- `ActivePopups.Num() > 0` → MappingContext 제거 (카메라 입력 차단)
- `ActivePopups.Num() == 0` → MappingContext 복원 (카메라 입력 재개)

### 장점

- CameraPawn → HUD 의존성 없음 (HUD가 단방향으로 제어)
- 다중 팝업 열림/닫힘 순서에 안전 (카운트 기반)
- 입력 핸들러 코드 수정 불필요

## 변경 내용

### CardGameCameraPawn.h

- `GetCameraMappingContext()` public getter 추가 — HUD에서 MappingContext 참조에 사용

### CardGameHUD.h

- `HasActivePopups()` public 쿼리 메서드 추가 — 다른 시스템에서도 활용 가능
- `UpdateCameraInput()` private 헬퍼 메서드 선언

### CardGameHUD.cpp

- `UpdateCameraInput()` 구현:
  - PlayerController → CameraPawn cast → MappingContext 획득 → InputSubsystem 제어
  - 각 단계 null 체크로 안전성 확보
- 호출 지점 3곳: `OpenPopup()`, `ClosePopup()`, `CloseAllPopups()` 끝
