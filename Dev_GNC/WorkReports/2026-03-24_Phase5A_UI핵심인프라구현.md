# Phase 5-A: UI 핵심 인프라 구현

> **날짜**: 2026-03-24
> **작업 유형**: 기능 구현 (Unity→UE5 포팅)
> **Phase**: Phase 5-A — UI 핵심 인프라

---

## 작업 요약

Unity C# 카드게임의 UIManager, ViewCore, PageView, PopupView, FloatingText를 UE5 C++ UMG 기반으로 포팅.
ACardGameHUD를 중심으로 페이지 스택 + 팝업 관리 + 플로팅 텍스트 풀링 시스템을 구축하였다.

CommonUI 미사용, 표준 UMG + 커스텀 뷰 스택 방식 채택.

## 주요 구현 상세

### 1. UViewBase (추상 위젯 베이스)
- 라이프사이클: `NativePreOpen(FViewParam)` → `NativeOnOpened()` → `NativePreClose()` → `NativeOnClosed()`
- BP 훅: `BP_OnOpened()`, `BP_OnClosed()` (BlueprintImplementableEvent)
- `RequestClose()`: OwningHUD에 닫기 위임 (페이지/팝업 자동 판별)

### 2. UPageWidget (페이지 베이스)
- `bKeepPreviousVisible`: true이면 다른 페이지가 위에 열려도 숨기지 않음 (Unity `_dontHide` 대응)
- 스택 관리는 ACardGameHUD가 담당

### 3. UPopupWidget (팝업 베이스)
- `PopupZOrder`: 팝업 간 Z순서 (높을수록 위)
- 다중 공존 가능

### 4. UFloatingTextWidget
- `Show(Type, Text, WorldPos)`: 월드 좌표 저장 + BP_PlayAnimation 호출
- `NativeTick`: 매 프레임 월드→스크린 좌표 프로젝션 + RemainingTime 감소
- Duration 1초 후 자동 숨김
- 뷰포트 스케일 보정 (`UWidgetLayoutLibrary::GetViewportScale`)

### 5. ACardGameHUD (핵심)

**A. 페이지 스택:**
- `OpenPage(TSubclassOf, FViewParam)` → 캐시 체크 → CreateWidget → PageLayer AddChild → 이전 페이지 Collapsed
- `ClosePage()` → 스택 Top Pop → RemoveFromParent → 이전 페이지 Visible 복원
- `CloseAllPages()` → 전체 스택 정리

**B. 팝업 관리:**
- `OpenPopup(TSubclassOf, FViewParam)` → CreateWidget → PopupLayer AddChild
- `ClosePopup(UPopupWidget*)` → 개별 닫기
- `CloseAllPopups()` → 전체 닫기

**C. 플로팅 텍스트 풀:**
- BeginPlay에서 10개 사전 생성 (Collapsed 상태)
- `ShowFloatingText(Type, Text, WorldPos)` → 라운드 로빈 인덱스로 재사용
- 최상위 뷰포트 ZOrder(100)에 표시

**D. RequestCloseView:**
- ViewBase::RequestClose()에서 호출
- Cast로 팝업/페이지 판별 → 적절한 Close 메서드 디스패치

### 6. 루트 위젯 구조
```
WBP_CardGameRoot (BP에서 수동 생성)
  ├── [CanvasPanel] PageLayer   ← 페이지 동적 추가/제거
  └── [CanvasPanel] PopupLayer  ← 팝업 동적 추가/제거
```
C++에서 `WidgetTree->FindWidget(FName)` 으로 참조 취득.

## 아키텍처 매핑

```
Unity                          →  UE5
──────────────────────────────────────────────────
ViewCore (abstract)            →  UViewBase (UUserWidget)
PageView                       →  UPageWidget (UViewBase)
PopupView                      →  UPopupWidget (UViewBase)
UIManager (Singleton)          →  ACardGameHUD (AHUD)
  ├ _canvasPage + _pageStack   →    PageLayer (UPanelWidget) + PageStack (TArray)
  ├ _canvasPopup + _dicPopup   →    PopupLayer (UPanelWidget) + ActivePopups (TArray)
  └ HudController              →    FloatingTextPool (TArray, 풀링)
FloatingText                   →  UFloatingTextWidget (UUserWidget)
ViewParam (Dictionary)         →  FViewParam (USTRUCT)
```

### 라이프사이클 매핑
```
Unity                    →  UE5
───────────────────────────────────
ViewCore.PreOpen()       →  UViewBase::NativePreOpen(FViewParam)
ViewCore.OnOpened()      →  UViewBase::NativeOnOpened() → BP_OnOpened()
ViewCore.PreClose()      →  UViewBase::NativePreClose()
ViewCore.OnClosed()      →  UViewBase::NativeOnClosed() → BP_OnClosed()
ViewCore.Close()         →  UViewBase::RequestClose() → HUD 위임
```

## 설계 결정 사항

| 항목 | 결정 | 근거 |
|------|------|------|
| CommonUI vs 커스텀 | 커스텀 뷰 스택 | 페이지 8+팝업 5 규모에 CommonUI 과도, 플러그인 미활성 상태 |
| FloatingText 방식 | 스크린 스페이스 풀링 | Z-오더 제어 용이, 10개 고정 풀 재사용 |
| EFloatingTextType 위치 | 기존 CardGameTypes.h 유지 | 이미 Phase 1에서 정의됨, 중복 방지 |
| 아이템 위젯 분리 여부 | PageWidget 내부 통합 | 파일 수 감소, Unity 아이템 클래스 대부분 가벼움 |
