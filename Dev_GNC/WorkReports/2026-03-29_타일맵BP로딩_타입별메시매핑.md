# 타일맵 BP 로딩 + 타입별 메시 매핑

- **일시**: 2026-03-29
- **분류**: 기능 구현 (그리드 시스템)
- **범위**: GridSubsystem BP 타일 스폰, SquareTile 타입별 메시 분기

---

## 배경

GridSubsystem의 `CreateTile()`에서 `ASquareTile::StaticClass()`를 하드코딩하여 스폰하고 있어 블루프린트 서브클래스(BP_SquareTile)를 사용할 수 없었다. 또한 `ETileType`(Ground/Dirt/Water/Blocked)이 존재하지만 타입에 따른 비주얼 분기가 없어 모든 타일이 동일하게 표시되는 문제가 있었다.

## 구현 내용

### 1. GridSubsystem — BP 타일 클래스 로딩

Subsystem은 CDO 기반 생성이므로 `EditAnywhere` 링크가 불가하여, `Initialize()`에서 문자열 경로로 BP 클래스를 로드하는 방식을 적용.

| 파일 | 변경 내용 |
|------|-----------|
| `GridSubsystem.h` | `TSubclassOf<ASquareTile> SquareTileClass` private 멤버 추가 |
| `GridSubsystem.cpp` | `Initialize()`에서 `LoadClass<ASquareTile>()` 호출, 실패 시 C++ 클래스 fallback |
| `GridSubsystem.cpp` | `CreateTile()`에서 `SquareTileClass`로 스폰하도록 변경 |

- BP 경로: `/Game/Variant_CardGame/Blueprints/Grid/BP_SquareTile.BP_SquareTile_C`
- 기존 프로젝트의 DataSubsystem/TextSubsystem과 동일한 `LoadObject`/`LoadClass` 패턴 준수

### 2. SquareTile — 타입별 메시 매핑

BP_SquareTile의 클래스 기본값에서 타입별 메시를 설정할 수 있도록 `TMap` 기반 매핑 추가.

| 파일 | 변경 내용 |
|------|-----------|
| `SquareTile.h` | `TMap<ETileType, TSoftObjectPtr<UStaticMesh>> TileMeshByType` (EditDefaultsOnly) 추가 |
| `SquareTile.h` | `ApplyTileTypeMesh()` private 메서드 선언 |
| `SquareTile.cpp` | `ApplyTileTypeMesh()` 구현 — TileMeshByType에서 현재 타입의 메시를 찾아 설정 |
| `SquareTile.cpp` | `Init()`, `SetTileType()`에서 `ApplyTileTypeMesh()` 호출 추가 |

- `TSoftObjectPtr<UStaticMesh>` 사용으로 에셋 지연 로딩 지원
- `EditDefaultsOnly`로 BP 기본값에서만 편집 가능 (인스턴스별 수정 방지)
