# Phase 5-D: 관리 페이지 구현

> **날짜**: 2026-03-25
> **작업 유형**: 기능 구현 (Unity→UE5 포팅)
> **Phase**: Phase 5-D — 관리 페이지 (덱 편성, 폰 관리, 상점, 모집)

---

## 작업 요약

Unity의 DeckSettingPage, PawnManagePage, ShopPage, RecruitPage를 UE5 C++로 포팅했다.
전제 조건으로 DPawn에 Equip/Unequip 메서드를 추가하고, 동적 리스트용 아이템 위젯 3종을 신규 생성한 뒤, 관리 페이지 4종을 구현하고 LobbyPageWidget의 스텁 핸들러를 실제 OpenPage 호출로 교체했다.

---

## 신규 파일 (14개)

### 1. UPawnListItemWidget (`UI/PawnListItemWidget.h/cpp`)
- UUserWidget 상속, UCLASS(Abstract)
- BindWidget: TextName, ButtonSelect
- SetData(UDPawn*), delegate FOnPawnItemClicked
- DeckSettingPage / PawnManagePage 공용 폰 목록 아이템

### 2. UEquipListItemWidget (`UI/EquipListItemWidget.h/cpp`)
- UUserWidget 상속, UCLASS(Abstract)
- BindWidget: TextName, TextSlotType, ButtonSelect
- SetData(UDEquipment*), delegate FOnEquipItemClicked
- EEquipSlotType → 표시명 매핑 포함

### 3. UShopCardItemWidget (`UI/ShopCardItemWidget.h/cpp`)
- UUserWidget 상속, UCLASS(Abstract)
- BindWidget: TextName, TextPrice, ButtonBuy
- SetData(UDCard*), delegate FOnShopBuyRequested
- 카드 이름 + 가격 표시, 구매 버튼

### 4. UDeckSettingPageWidget (`UI/DeckSettingPageWidget.h/cpp`)
- UPageWidget 상속, UCLASS(Abstract)
- **Pawn 탭**: 고정 슬롯 5개 (ButtonSlot0~4, TextSlot0~4) + ScrollOwnedPawns
  - 잠긴 슬롯 클릭: 100골드 → ExpandSlot
  - 빈 슬롯 + 선택된 폰: AssignPawnToDeck
  - 폰 있는 슬롯: UnassignPawnFromDeck
  - 편성되지 않은 폰만 스크롤에 동적 생성
- **Card 탭**: ScrollDeckCards에 TextBlock으로 카드 목록 표시 (읽기 전용)
- 서브시스템: DeckSubsystem, PawnSubsystem, PlayerDataSubsystem

### 5. UPawnManagePageWidget (`UI/PawnManagePageWidget.h/cpp`)
- UPageWidget 상속, UCLASS(Abstract)
- **좌측**: ScrollPawnList (폰 목록, 클릭 선택)
- **우측**: PanelPawnInfo (스탯 6종 + 장비 슬롯 4개)
  - 슬롯 타입: [0]=WeaponMain, [1]=Armor, [2]=Tool, [3]=Tool
  - 장비 있으면 클릭=Unequip, 없으면+SelectedEquip=Equip (슬롯 타입 검증)
  - OnStatsChanged 구독으로 스탯 자동 갱신
- **하단**: ScrollUnequippedList (미장착 장비만 필터링)
- FindEquipInSlot: 슬롯 인덱스 → 해당 타입의 N번째 장비 탐색
- 서브시스템: PawnSubsystem, ItemSubsystem

### 6. UShopPageWidget (`UI/ShopPageWidget.h/cpp`)
- UPageWidget 상속, UCLASS(Abstract)
- ScrollShopCards: ShopCardItemWidget 동적 생성
- ButtonLevelUp: 50골드 → LevelUpShop → GenerateShopList
- HandleBuyCard: Price 골드 차감 → DeckSubsystem.AddCard → RemoveShopCard
- OnShopListChanged 델리게이트 구독
- 서브시스템: LobbySubsystem, PlayerDataSubsystem, DeckSubsystem

### 7. URecruitPageWidget (`UI/RecruitPageWidget.h/cpp`)
- UPageWidget 상속, UCLASS(Abstract)
- 고정 슬롯 3개 (PanelRecruit0~2, ButtonRecruit0~2, TextRecruitName0~2)
  - 100골드 → PawnSubsystem.AddPawn → LobbySubsystem.RemoveRecruitPawn
  - 빈 슬롯 Collapsed 처리
- ButtonReroll: 50골드 → GenerateRecruitList
- OnRecruitListChanged 델리게이트 구독
- 서브시스템: LobbySubsystem, PawnSubsystem, PlayerDataSubsystem

## 수정 파일 (4개)

### 8. UDPawn (`Data/DPawn.h/cpp`)
- `Equip(UDEquipment*)`: Equips.Add → SetEquippedPawn → RecalculateStats
- `Unequip(UDEquipment*)`: Equips.Remove → ClearEquippedPawn → RecalculateStats
- bool 반환으로 실패 처리 가능

### 9. UDeckSubsystem (`Subsystem/DeckSubsystem.h`)
- `GetDeckCards()` getter 추가 (private DeckCardList 접근용, 인라인)

### 10. ULobbyPageWidget (`UI/LobbyPageWidget.h/cpp`)
- 페이지 클래스 프로퍼티 4개 추가 (EditDefaultsOnly):
  - DeckSettingPageClass, PawnManagePageClass, ShopPageClass, RecruitPageClass
- 4개 핸들러: UE_LOG 스텁 → `OwningHUD->OpenPage(XxxPageClass)` 교체

## Unity → UE5 매핑

| Unity | UE5 |
|-------|-----|
| DeckSettingPage (탭: Pawn/Card) | UDeckSettingPageWidget (PanelPawnTab/PanelCardTab) |
| DeckSetting_Pawn (5슬롯 + 드래그-드롭) | ButtonSlot0~4 클릭 방식 (드래그-드롭 미구현) |
| DeckSetting_Card (스크롤 그리드) | ScrollDeckCards TextBlock 목록 |
| PawnManagePage (좌: 폰, 우: 스탯+장비) | UPawnManagePageWidget 동일 구조 |
| DPawn.Equip/Unequip | UDPawn::Equip/Unequip (RecalculateStats 포함) |
| PawnManageEquipSlot (드래그-드롭) | ButtonEquipSlot0~3 클릭 방식 |
| ShopPage (카드 구매 + LevelUp) | UShopPageWidget + ShopCardItemWidget |
| RecruitPage (폰 모집 + Reroll) | URecruitPageWidget (고정 3슬롯) |
| DeckPawnItem / PawnManageItem | UPawnListItemWidget (공용) |
| PawnManageEquipItem | UEquipListItemWidget |
| ShopItem | UShopCardItemWidget |
