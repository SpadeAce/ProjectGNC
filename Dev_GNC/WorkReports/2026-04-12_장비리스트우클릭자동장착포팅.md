# 장비 리스트 우클릭 자동 장착 기능 포팅

- **일시**: 2026-04-12
- **분류**: 기능 포팅 (UI / 장비 시스템)
- **범위**: EquipListItemWidget.h/.cpp, PawnManagePageWidget.h/.cpp

---

## 배경

Unity 원본의 PawnManagePage에서는 미장착 장비 리스트 아이템을 우클릭하면 선택된 폰의 매칭 슬롯에 자동 장착되는 기능이 있었다. UE5 포팅 시 좌클릭 선택 → 슬롯 버튼 클릭의 2단계 장착 흐름만 구현되었고, 우클릭 자동 장착 기능이 누락되어 있었다.

## Unity 원본 동작

- `PawnManageEquipItem.OnPointerClick()`: 우클릭 감지 → `OnRightClick` 이벤트 발생
- `PawnManagePage.OnEquipItemRightClick()`: 장비의 슬롯 타입과 매칭되는 빈 슬롯 우선 탐색, 없으면 첫 번째 매칭 슬롯에 교체 장착
- `EquipToSlot()`: 기존 장비 해제 → 새 장비 장착 → UI 갱신

## 구현 내용

### 1. EquipListItemWidget — 우클릭 감지

- `FOnEquipItemRightClicked` 다이나믹 멀티캐스트 델리게이트 추가
- `NativeOnMouseButtonDown` 오버라이드: 우클릭(`RightMouseButton`) 시 `bRightMouseDown = true`로 마킹, `FReply::Handled()` 반환
- `NativeOnMouseButtonUp` 오버라이드: 우클릭 + `bRightMouseDown`이면 `OnItemRightClicked.Broadcast(Equip)` 호출
- 좌클릭은 기존 `ButtonSelect->OnClicked` 흐름 유지 (충돌 없음)

### 2. PawnManagePageWidget — 바인딩 및 자동 장착 로직

- `NativePreOpen`: `TileUnequippedList->OnEntryWidgetGenerated()`/`OnEntryWidgetReleased()`에 바인딩
- `HandleEntryWidgetGenerated`: 위젯을 `UEquipListItemWidget`으로 캐스트 후 `OnItemRightClicked.AddDynamic` 바인딩
- `HandleEntryWidgetReleased`: `OnItemRightClicked.RemoveDynamic` 해제 (풀링 재사용 시 중복 바인딩 방지)
- `HandleEquipItemRightClicked`: 선택된 폰 확인 후 `AutoEquipToSlot` 호출
- `AutoEquipToSlot`: Unity `OnEquipItemRightClick` + `EquipToSlot` 로직 포팅
  1. `SlotTypeMap`으로 장비 타입에 매칭되는 슬롯 인덱스 수집
  2. `FindEquipInSlot() == nullptr`인 빈 슬롯 우선 탐색
  3. 빈 슬롯 없으면 첫 번째 매칭 슬롯의 기존 장비 `Pawn->Unequip()` 후 교체
  4. `Pawn->Equip()` → `SelectedEquip` 초기화 → UI 갱신 3종 호출

## 변경 파일

| 파일 | 변경 사항 |
|------|-----------|
| `EquipListItemWidget.h` | 델리게이트 선언, 오버라이드 선언, `bRightMouseDown` 멤버 |
| `EquipListItemWidget.cpp` | `NativeOnMouseButtonDown`/`Up` 구현 |
| `PawnManagePageWidget.h` | `HandleEquipItemRightClicked`, `AutoEquipToSlot`, `HandleEntryWidgetGenerated`/`Released` 선언 |
| `PawnManagePageWidget.cpp` | 위 4개 함수 구현, `NativePreOpen`/`PreClose`에 바인딩/해제 추가 |

## 설계 포인트

- **TileView 풀링 대응**: `OnEntryWidgetGenerated`/`Released`를 사용하여 위젯이 생성/해제될 때 우클릭 델리게이트를 동적으로 바인딩/해제. TileView의 위젯 재사용 메커니즘과 충돌 없음
- **기존 코드 재사용**: `FindEquipInSlot()`, `SlotTypeMap[]`, `Pawn->Equip()`/`Unequip()`, `Refresh*()` 메서드를 그대로 활용
- **Down/Up 쌍 처리**: `NativeOnMouseButtonDown`에서 `FReply::Handled()` 반환해야 `NativeOnMouseButtonUp`이 호출됨. 드래그 등 오동작 방지를 위해 `bRightMouseDown` 플래그 사용
