# CardGameCameraPawn 카메라 Yaw 및 Pan 방향 수정

- **일시**: 2026-04-06
- **분류**: 버그 수정 (카메라)
- **범위**: CardGameCameraPawn.h/.cpp

---

## 배경

Unity 원본 `CameraController.cs`에서는 카메라가 초기 Yaw 45도로 시작하고, 우클릭 드래그 시 "맵을 잡고 끄는" 방향으로 패닝이 동작한다. UE5 포팅 과정에서 두 가지가 누락되어 카메라가 어색하게 동작하고 있었다:

1. 초기 Yaw가 0도로 시작 → 45도 꺾임이 없어 시점이 어색함
2. 드래그 방향이 반대 → 마우스를 오른쪽으로 끌면 화면이 반대로 이동

---

## 변경 내용

### 1. InitialYaw 프로퍼티 추가 (CardGameCameraPawn.h)

```cpp
UPROPERTY(EditDefaultsOnly, Category = "CardGame|Camera")
float InitialYaw = 45.0f;
```

Unity `_initialYaw = 45f`에 대응하는 프로퍼티 추가.

### 2. BeginPlay에서 InitialYaw 적용 (CardGameCameraPawn.cpp)

```cpp
// 변경 전
TargetYaw = GetActorRotation().Yaw;
CurrentYaw = TargetYaw;

// 변경 후
TargetYaw = InitialYaw;
CurrentYaw = InitialYaw;
```

### 3. HandlePan 방향 반전 (CardGameCameraPawn.cpp)

```cpp
// 변경 전
const FVector Delta = (Forward * Input.Y + Right * Input.X) * PanSpeed;

// 변경 후
const FVector Delta = -(Forward * Input.Y + Right * Input.X) * PanSpeed;
```

Unity 원본에서 `-` 부호로 역방향 패닝을 구현한 것과 동일하게 적용.
