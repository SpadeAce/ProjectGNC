# Phase 5-E: 팝업 UI 시스템 구현

> **날짜**: 2026-03-26
> **작업 유형**: 기능 구현 (Unity→UE5 포팅)
> **Phase**: Phase 5-E — 팝업 위젯 (일시정지, 범용 확인, 전투 결과)

## 작업 요약

Unity의 PausePopup, NoticePopup, BattleResultPopup을 UE5 C++로 포팅했다.
Phase 5-A에서 구축한 PopupWidget 베이스 클래스와 CardGameHUD의 팝업 관리 인프라(OpenPopup/ClosePopup)를 활용하여 팝업 3종을 구현하고, StagePageWidget의 기존 스텁 핸들러를 실제 팝업 호출로 교체했다.

---

## 신규 파일 (6개)

### 1. UPausePopupWidget (`UI/PausePopupWidget.h/cpp`)
- UPopupWidget 상속, UCLASS(Abstract)
- BindWidget: ButtonContinue, ButtonOption, ButtonExit
- Continue → RequestClose()로 자기 자신 닫기
- Option → 스텁 (OptionPopup 미구현, 로그 출력)
- Exit → CloseAllPopups + CloseAllPages (Phase 6에서 SceneController 연결 예정)
- Unity PausePopup.cs 대응

### 2. UNoticePopupWidget (`UI/NoticePopupWidget.h/cpp`)
- UPopupWidget 상속, UCLASS(Abstract)
- BindWidget: TextTitle, TextBody, ButtonConfirm, ButtonCancel
- 델리게이트 콜백 패턴: FOnNoticeConfirm / FOnNoticeCancel (DECLARE_DELEGATE)
- SetContent(FText Title, FText Body)로 텍스트 설정
- 확인/취소 클릭 시 콜백 실행 후 자동 닫기
- NativePreClose에서 델리게이트 Unbind 처리
- Unity NoticePopup.cs 대응

### 3. UBattleResultPopupWidget (`UI/BattleResultPopupWidget.h/cpp`)
- UPopupWidget 상속, UCLASS(Abstract)
- BindWidget 11개: TextTitle, TextRewardGold, TextRewardExp, ButtonContinue, ButtonExit, RewardCardPanel, ButtonRewardCard0~2, TextRewardCard0~2
- FViewParam 매핑: bFlag=승리여부, IntValue=골드보상, ExpReward는 StageSubsystem에서 직접 조회
- 승리/패배 분기: 버튼 표시 전환, 보상 카드 영역 표시/숨김
- 가중치 기반 중복 없는 카드 3장 선택 (PickWeightedCards)
- 카드 토글 선택: 같은 카드 재클릭 시 해제, 다른 카드 클릭 시 전환
- RenderOpacity로 선택 시각 피드백 (선택=1.0, 비선택=0.6)
- Continue 클릭 시: 골드 추가, 난이도 증가, 편성 폰 전원 경험치, 선택 카드 덱 추가
- Exit 클릭 시: 전체 UI 닫기 (Phase 6에서 씬 전환 연결 예정)
- Unity BattleResultPopup.cs 대응

## 설계 결정

### NoticePopup 콜백 패턴
- Unity는 `Action` 델리게이트를 Init() 파라미터로 전달
- UE5에서는 FViewParam이 UObject만 지원하므로, **팝업 인스턴스의 퍼블릭 델리게이트 프로퍼티**로 전환
- 호출 측에서 OpenPopup 후 OnConfirm/OnCancel을 직접 바인딩하는 2단계 패턴 채택

### BattleResultPopup 데이터 전달
- FViewParam의 bFlag/IntValue로 승리여부와 골드 전달
- ExpReward는 별도 UObject 없이 NativePreOpen에서 StageSubsystem 직접 접근
- 보상 카드 후보는 DataSubsystem→GetStageRewardByLevel()로 조회

### 보상 카드 선택 UI
- 별도 아이템 위젯 클래스 없이 팝업 내부에서 Button+TextBlock 3세트로 처리
- 선택 상태는 RenderOpacity로 시각 구분 (BP에서 스타일 커스터마이징 가능)

### 씬 전환 스텁
- PausePopup의 Exit와 BattleResultPopup의 Continue/Exit에서 씬 전환이 필요하나, Phase 6(씬 흐름) 전까지는 CloseAllPopups + CloseAllPages로 대체
