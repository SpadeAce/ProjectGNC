# 작업 보고서: DrawCard 효과 구현 및 카드 비활성화 버그 수정

**날짜**: 2026-03-06
**작업자**: Claude

## 요청 내용

`DRAW_CARD` 카드 이펙트를 구현하여, 해당 효과가 있는 카드를 사용하면 EffectValue만큼 핸드에서 드로우하는 기능을 추가하였다. 또한 카드 사용 후 비활성화 상태가 갱신되지 않는 버그를 수정하였다.

## 수행한 작업

- `DeckManager.DrawCards(int count)` 접근자를 `private` → `public`으로 변경
- `StageManager.ApplyCardEffects()`의 switch-case에 `CardEffectType.DrawCard` 케이스 추가 → `DeckManager.Instance.DrawCards(value)` 호출
- 카드 비활성화 갱신 버그 수정: `StageManager`의 `UseCardSelf` 및 `TryUseCard`에서 `ConsumeActingPower`를 `DeckManager.UseCard` 이전에 호출하도록 순서 변경

## 수정된 파일

| 파일 경로 | 변경 유형 | 변경 내용 요약 |
|-----------|-----------|----------------|
| `Assets/Scripts/Logic/Manager/DeckManager.cs` | 수정 | DrawCards 메서드 private → public |
| `Assets/Scripts/Logic/Manager/StageManager.cs` | 수정 | ApplyCardEffects에 DrawCard case 추가, ConsumeActingPower 호출 순서 수정 |

## 주요 결정 사항

- **DrawCards public 노출**: 별도 wrapper 없이 기존 메서드를 직접 public으로 변경 — MaxHandSize 초과 방지, 소모풀 리필 등 기존 로직이 이미 완비되어 있어 재사용이 적합
- **버그 원인**: `UseCard()` → `onHandChanged` → `SetCardList()` 순서로 실행될 때 아직 `ConsumeActingPower()`가 호출되지 않아 이전 ActingPower 값으로 disabled 계산. `ConsumeActingPower`를 먼저 호출하면 `SetCardList()` 실행 시점에 이미 차감된 값을 참조하여 해결

## 참고 사항

- `CardEffectType.DrawCard`는 GameData.cs에 값 6으로 이미 정의되어 있었음 (프로토버퍼 생성 파일, 수정 불필요)
- 기획서 `03_카드_시스템.md`의 CardEffectType 표에 DrawCard 항목이 누락되어 있었음 — 별도로 갱신
