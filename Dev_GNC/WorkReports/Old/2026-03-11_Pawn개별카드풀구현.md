# 작업 보고서: Pawn 개별 카드풀 구현

**날짜**: 2026-03-11
**작업자**: Claude

## 요청 내용

Stage에서 사용할 카드 시스템을 개선한다. Pawn의 핸드를 공용 카드풀이 아닌 자신이 장착한 장비(`EquipmentData.CardId`)를 기반으로 동작하도록 변경하고, Pawn 카드 규칙을 초기 2장 / 턴당 1장 드로우 / 최대 3장으로 수정한다.

## 수행한 작업

- `DPawn`에 `_pawnCardPool`, `_pawnDiscardPool` 필드 및 카드풀 관련 메서드 추가 (`BuildCardPool`, `InitHand`, `DrawCards`, `UsePawnCard`, `RefillPawnFromDiscard`, `ShufflePawnPool`)
- `DPawn`에 상수 `InitialHandSize = 2`, `MaxHandSize = 3`, `DrawPerTurn = 1` 정의
- `DeckManager`에서 Pawn 전용 공용풀 로직 제거 (`InitialPawnHandSize`, `MaxPawnHandSize`, `PawnDrawPerTurn` 상수, `InitPawnHand`, `DrawPawnCards`, `UsePawnCard` 메서드)
- `DeckManager.InitStage`에서 `pawn.PawnHandCards.Clear()` 루프 제거 (BuildCardPool에서 처리)
- `StageScript.OnEnterScene`에서 `DeckManager.InitPawnHand` 호출을 `pawn.BuildCardPool() + pawn.InitHand()`로 교체
- `TurnManager.StartPawnTurn`에서 `DeckManager.DrawPawnCards` 호출을 `drawPawn.DrawCards(DPawn.DrawPerTurn)`으로 교체
- `StageManager`의 `UseCardSelf` / `TryUseCard` 4곳에서 `DeckManager.UsePawnCard(card, cardOwner)` 호출을 `cardOwner.UsePawnCard(card)`로 교체
- `05_캐릭터_시스템.md`에 Pawn 개별 카드풀 섹션 추가 및 향후 작업 체크리스트 갱신
- `06_UI_시스템.md`의 StagePage 항목에 Pawn 카드 UI 상세 내용 추가

## 주요 결정 사항

- **카드풀 관리를 DPawn으로 이전**: 공용풀 의존성을 완전히 끊고 Pawn이 자신의 풀을 자체 관리하도록 설계. StageManager / TurnManager 호출 지점만 변경해 외부 인터페이스를 최소화.
- **소모풀(discard) 역시 Pawn 소속으로**: 기존에는 사용한 Pawn 카드가 DeckManager의 공용 `_discardPool`로 유입되어 글로벌 핸드에 섞이는 문제가 있었음. Pawn별 `_pawnDiscardPool`로 완전 분리.
- **상수를 `DPawn`의 `public const`로 정의**: TurnManager 등 외부에서 `DPawn.DrawPerTurn`으로 참조 가능하도록 하여 수치 변경 시 단일 지점만 수정하면 됨.

## 참고 사항

- `InitialHandSize`, `MaxHandSize`는 추후 DPawn의 스탯 시스템과 연동하여 장비 보너스로 확장 예정.
- 글로벌 핸드(`DeckManager.handCardList`, `_goCardGroup`)는 현재도 유지되며 별도로 동작함.
- 장비 미장착 Pawn은 카드풀이 비어 핸드가 0장으로 시작한다.
