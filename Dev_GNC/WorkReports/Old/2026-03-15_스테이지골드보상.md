# 작업 보고서: 스테이지 골드 보상

**날짜**: 2026-03-15
**작업자**: Claude

## 요청 내용

`MonsterData`에 추가된 `Gold` 필드를 활용해, 몬스터 처치 시 해당 값만큼 골드를 누적하고 스테이지 클리어 시 누적 골드 + 100(클리어 보너스)을 `PlayerManager`에 지급하는 보상 시스템을 구현.

## 수행한 작업

- `StageManager`에 `_stageGoldReward` 필드 및 `StageGoldReward` 프로퍼티 추가
- `ResetStageState()`에서 `_stageGoldReward = 0` 초기화 추가
- `ApplyCardEffects()` 내 몬스터 사망 분기에서 `_stageGoldReward += monster.Data.Gold` 누적 처리 추가
- `CheckBattleResult()`에서 전원 적 전멸(승리) 시 클리어 보너스 100 추가 후 `onBattleEnd` 발생
- `BattleResultPopup.BattleResultParam`에 `goldReward` 필드 추가
- `BattleResultPopup.OnClickContinue()`에서 `PlayerManager.Instance.AddGold(goldReward)` 호출
- `StagePage.OnBattleEnd()`에서 승리 시 `StageManager.Instance.StageGoldReward`를 `BattleResultParam`에 전달

## 수정된 파일

| 파일 경로 | 변경 유형 | 변경 내용 요약 |
|-----------|-----------|----------------|
| `Assets/Scripts/Logic/Manager/StageManager.cs` | 수정 | `_stageGoldReward` 누적 로직, 클리어 보너스 +100, 초기화 |
| `Assets/Scripts/UI/BattleResult/BattleResultPopup.cs` | 수정 | `BattleResultParam.goldReward` 필드 추가, `AddGold()` 호출 |
| `Assets/Scripts/UI/Stage/StagePage.cs` | 수정 | `OnBattleEnd()`에서 `goldReward` 전달 |

## 주요 결정 사항

- **누적 위치를 StageManager로**: 골드 누적은 전투 로직 레이어(`StageManager`)에서 처리하고, 실제 지급은 씬 전환 시점(`BattleResultPopup.OnClickContinue()`)에서 수행. 전투 중 즉시 지급하지 않아 패배 시 보상이 반영되지 않음.
- **클리어 보너스는 CheckBattleResult()에서 선 적용**: 보너스 100은 `StageGoldReward`에 포함시켜 단일 값으로 전달. 별도 필드로 분리하지 않음.
- **패배 시 goldReward = 0**: `StagePage.OnBattleEnd()`에서 `isWin`이 false이면 0을 전달하여 패배 시 골드 미지급.

## 참고 사항

- `MonsterData.Gold` 값이 0인 몬스터는 처치해도 골드가 누적되지 않음 (의도된 동작).
- 향후 골드 보상 UI 표시(BattleResultPopup에 획득 골드 수치 표기) 추가 가능.
