# 작업 보고서: 플로팅 텍스트 시스템

**날짜**: 2026-03-18
**작업자**: Claude

## 요청 내용

전투 중 데미지, 회복, 실드, 버프/디버프 등 수치 변화를 액터 위에 떠오르는 텍스트로 표시하는 공용 플로팅 텍스트 시스템을 구현하고, 종류별로 다른 연출(데미지: 대각 포물선 바운스, 회복: 수직 상승, 버프/디버프: 제자리 팝)을 적용하도록 요청받았다.

## 수행한 작업

- `FloatingText.cs`를 생성하여 `FloatingTextType` enum과 `MonoBehaviourEx` 기반 플로팅 텍스트 컴포넌트를 구현했다.
- 종류별 연출 코루틴 3종을 구현했다: `DamageBounce`(대각 포물선 + 바운스), `VerticalRise`(수직 상승 + 페이드아웃), `PopInPlace`(제자리 스케일 팝 + 페이드아웃).
- `HudController.cs`를 생성하여 FloatingText 오브젝트 풀(초기 10개, 자동 확장)과 액터별 Y축 오프셋 누적 겹침 방지 로직을 구현했다.
- `UIManager.cs`에 `_hudController` SerializeField와 `Hud` 프로퍼티를 추가하고, `Awake()`에서 초기화, `CloseAllView()`에서 `ClearAll()` 호출을 추가했다.
- `DPawn.cs`에 `onFloatingText` 이벤트를 추가하고, `TakeDamage`, `Heal`, `AddShield`, `ApplyBuff`에서 해당 이벤트를 호출하도록 수정했다.
- `DMonster.cs`에 동일하게 `onFloatingText` 이벤트를 추가하고 각 메서드에서 호출하도록 수정했다.
- `Actor.cs`의 `Init(DPawn)`, `Init(DMonster)`에서 `onFloatingText` 이벤트를 구독하고, `OnFloatingText` 핸들러에서 `UIManager.Instance.Hud.ShowFloatingText()`를 호출하도록 수정했다.
- 기획서 3건을 갱신했다 (06_UI_시스템, 02_전투_시스템, 99_로드맵).

## 주요 결정 사항

- **UIManager 하위 HudController 방식 채택**: 별도 FloatingTextManager(MonoSingleton)를 만들지 않고, UIManager 프리팹의 자식 오브젝트로 HudController를 배치했다. HUD 관련 기능을 UIManager 산하에서 통합 관리하여 매니저 증가를 방지한다.
- **MonoBehaviourEx 상속**: FloatingText를 MonoBehaviourEx로 구현하여 LinkerAttribute 기반 자동 연결 기능을 활용 가능하게 했다.
- **AssetPath 기반 프리팹 로드**: HudController에 SerializeField로 프리팹을 받지 않고, FloatingText에 `[AssetPath]`를 부여하여 `PrefabLoader.Load<FloatingText>()`로 로드한다. 불필요한 Inspector 연결을 제거했다.
- **DPawn/DMonster 이벤트 기반**: 기존 `onStatsChanged`와 별도로 `onFloatingText(FloatingTextType, int)` 이벤트를 추가하여, 카드 효과와 몬스터 일반공격 모두 자동으로 플로팅 텍스트가 표시되도록 했다.
- **종류별 차별화 연출**: Damage는 대각 포물선 + 바운스, Heal/Shield는 수직 상승, Buff/Debuff/Block은 제자리 스케일 팝으로 시각적 구분을 명확히 했다.
