# 작업 보고서: 골드 시스템 구현

**날짜**: 2026-03-08
**작업자**: Claude

## 요청 내용

상점과 영입 화면에서 사용할 재화(골드) 시스템 구현 요청.
초기 골드 1000으로 세팅하고, 상점 카드 구매 시 CardData.Price만큼, Pawn 영입 시 100(임시 고정)만큼 차감하는 기능 구현.

## 수행한 작업

- `DeckManager`에 골드 상태(`_gold = 1000`)와 `Gold` 프로퍼티, `SpendGold()`, `AddGold()` 메서드 추가
- `RecruitItem`의 가격 표시를 `"FREE"` → `100`으로 변경하고 `RecruitPrice = 100` 상수 정의
- `RecruitItem.OnClickRecruit()`에 `SpendGold()` 체크 추가 — 잔액 부족 시 영입 불가
- `ShopItem.OnClickBuy()`에 `SpendGold()` 체크 추가 — 잔액 부족 시 구매 불가
- `08_자원_시스템.md` 기획서에 Gold 섹션(3번) 추가 및 핵심 파일 표/향후 작업 갱신
- `09_상점_시스템.md` 골드 시스템 연동 항목 완료 처리
- `99_로드맵.md` 골드 시스템 연동 항목 완료 처리

## 주요 결정 사항

- **골드 위치를 DeckManager에 배치**: 별도 Manager 클래스 신설 없이 기존 플레이어 상태 관리 싱글톤에 통합. 현재 요구 범위(차감만)에서는 over-engineering 없이 가장 단순한 방법.
- **잔액 부족 시 return 처리**: 버튼 비활성화 대신 클릭 시 체크 후 무시. 요청 범위가 "차감 기능만"이었으므로 최소 구현 유지. 버튼 비활성화는 Gold UI 표시 구현 시 함께 추가 예정.
- **영입 가격 하드코딩(100)**: PawnData에 가격 필드가 없으므로 `RecruitItem.RecruitPrice` 상수로 임시 처리. 데이터화는 향후 작업으로 분류.

## 참고 사항

- Gold UI 표시(로비 화면에 현재 골드 수치) 미구현 — `08_자원_시스템.md` 향후 작업에 등록.
- Gold 획득 경로 미구현 — 스테이지 클리어 보상 등 연동 시 `DeckManager.AddGold()` 호출.
- Pawn 영입 가격은 추후 PawnData에 Price 필드 추가 후 데이터화 필요.
