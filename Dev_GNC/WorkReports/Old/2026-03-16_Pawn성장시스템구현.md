# 작업 보고서: Pawn 성장 시스템 구현

**날짜**: 2026-03-16
**작업자**: Claude

## 요청 내용

MonsterData의 exp 필드를 통해 경험치를 획득하고, 스테이지 종료 시 참여 Pawn에게 일괄 지급하는 성장 시스템을 구현해달라는 요청.
레벨업 시 PawnGrowthData에 정의된 스탯 보너스와 카드를 영구 획득하는 구조 포함.

## 수행한 작업

- `PawnGrowthTable` 등록 (`/add-table PawnGrowthDataTable` 커맨드로 DataManager에 추가)
- `PawnGrowthTable.cs`에 `GetByClassLevel(ClassType, level)` 메서드 추가
- `PawnStats.Recalculate()`에 `growthBonuses` 파라미터 추가하여 성장 스탯 합산 지원
- `DPawn`에 `Level(=0)`, `Exp`, `_growthBonuses`, `_growthCardIds` 필드 추가
- `DPawn.AddExp()` / `TryLevelUp()` 메서드 구현 (다중 레벨업 while 루프 포함)
- `DPawn.BuildCardPool()`에 성장 카드(`_growthCardIds`) 포함 처리 추가
- `DPawn.RecalculateStats()` 및 생성자에 `growthBonuses` 전달 추가
- `StageManager`에 `_stageExpReward` 필드 추가, 몬스터 처치 시 누적 및 ResetStageState에서 초기화
- `StagePage.OnBattleEnd()`에서 `expReward`를 `BattleResultParam`에 전달
- `BattleResultPopup.BattleResultParam`에 `expReward` 필드 추가
- `BattleResultPopup.OnClickContinue()`에서 `DeckManager.DeckPawns` 순회하여 `pawn.AddExp()` 호출
- `05_캐릭터_시스템.md` 기획서 갱신 (성장 시스템 섹션 추가, 파일 목록/스탯 합산 구조/카드풀 설명 업데이트)
- `99_로드맵.md` 경험치 획득 연동 항목 완료 처리

## 주요 결정 사항

- **기본 레벨 0**: 레벨은 0에서 시작. PawnGrowthData의 첫 번째 항목은 Level=1로 설정하여 레벨업 시 조회됨.
- **경험치 분배 방식**: 스테이지 총 획득 경험치를 참여 Pawn 전원에게 동일하게 지급 (나누지 않음). Gold 지급 패턴과 동일한 구조.
- **성장 보너스 영구 보존**: `_growthBonuses` / `_growthCardIds`는 DPawn 인스턴스에 누적. 스테이지 간 유지되며 세이브 연동 시 직렬화 대상.
- **카드 반영 시점**: 성장 카드는 레벨업 즉시 `_growthCardIds`에 추가되고, 다음 스테이지 `BuildCardPool()` 호출 시 카드풀에 반영됨.
- **스탯 반영 시점**: 성장 스탯 보너스는 레벨업 시 `RecalculateStats()` 호출로 즉시 기저 스탯에 반영됨.

## 참고 사항

- `RequireExp`는 누적 경험치 기준. 예: Level 2 RequireExp=100이면 Exp가 100 이상일 때 레벨업.
- 레벨업 연출 UI(팝업, 애니메이션), 로비에서의 레벨/경험치 표시, 세이브 데이터 연동은 미구현 상태로 로드맵에 등록됨.
