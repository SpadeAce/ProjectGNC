# 작업 보고서: 버프디버프 시스템 구현

**날짜**: 2026-03-07
**작업자**: Claude

## 요청 내용

`CardEffectType`에 정의된 BuffAttack~DebuffMovement(7~12)를 구현해달라는 요청. 카드 효과 적용 시 대상의 스탯(Attack, Armor, Movement)을 EffectValue만큼 변경하고, Pawn의 턴이 시작될 때마다 Duration을 감소시켜 만료 시 원래 값으로 복귀. **사용한 턴도 지속시간에 포함** — Duration=2는 획득한 턴 포함 2턴 동안 유지, 3턴 시작 시 제거.

## 수행한 작업

- `BuffEntry.cs` 신규 생성 — `CardEffectType Type`, `int Value`(델타), `int RemainingTurns` 필드를 가지는 struct 정의
- `DPawn.cs`에 `_buffs: List<BuffEntry>`, `ApplyBuff()`, `TickBuffs()`, `ApplyStatDelta()` 추가
- `DMonster.cs`에 동일한 버프 관리 메서드 추가 (`using System.Collections.Generic` 추가 포함)
- `TurnManager.StartPawnTurn()`에 Pawn 루프에 `pawn.TickBuffs()` 추가, EnemyActors 루프 신규 추가하여 `mon.TickBuffs()` 호출
- `StageManager.ApplyCardEffects()`에 BuffAttack/BuffArmor/BuffMovement/DebuffAttack/DebuffArmor/DebuffMovement 6개 case 추가
- `StageManager.GetActorMovement()`의 `pawn.Data.Movement` → `pawn.Movement` 수정 (이동 버프가 실제 이동에 반영되지 않는 버그 수정)
- `WorkReports/Design/03_카드_시스템.md` 갱신 — CardEffectType 표 업데이트, 버프/디버프 시스템 섹션 추가
- `WorkReports/Design/99_로드맵.md` 갱신 — Buff/Debuff 관련 항목 전부 완료 처리

## 주요 결정 사항

- **직접 스탯 수정 방식 채택**: 버프 적용 시 `Attack += value`, 만료 시 `Attack -= value`로 필드를 직접 변경. 별도 "유효 스탯 계산" 레이어 없이 기존 패턴(Heal, AddShield 등)과 동일한 방식.
- **사용한 턴 포함 카운트**: `TickBuffs()`는 먼저 감소(decrement first) 후 0 이하이면 제거. Duration=2이면 획득 포함 2턴 유지, 3턴 시작에 제거.
- **Debuff는 음수 value로 통일**: `ApplyBuff(type, -value, duration)`으로 호출 — `ApplyStatDelta`는 Buff*/Debuff* 타입을 동일 case로 처리하여 코드 중복 최소화.
- **GetActorMovement 버그 수정 포함**: 기존에 `pawn.Data.Movement`(기본값)를 읽어 이동 버프가 실제 이동에 반영되지 않던 문제를 `pawn.Movement`(현재값)로 수정.

## 참고 사항

- Buff*는 보통 DPawn 대상(아군), Debuff*는 DMonster 대상(적)이지만 코드는 양쪽 모두 허용하도록 구현되어 있어 향후 설계 변경에 유연하게 대응 가능.
- 스탯 최솟값 0 클램프 적용 — 디버프로 인해 Attack/Armor/Movement가 음수가 되지 않음.
- 버프 중복 적용 시 각각 독립된 `BuffEntry`로 관리되므로 개별 만료 처리됨.
