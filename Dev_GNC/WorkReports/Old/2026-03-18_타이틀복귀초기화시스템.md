# 작업 보고서: 타이틀 복귀 초기화 시스템

**날짜**: 2026-03-18
**작업자**: Claude

## 요청 내용

타이틀로 돌아갈 때 Lobby/Stage에서 사용하던 모든 런타임 데이터를 초기화하는 전용 함수를 구현하고, 각 매니저에 공통 초기화 인터페이스를 추가하며, DeckManager.InitTestData()를 Boot에서 Lobby 진입 시점으로 이동하는 작업을 요청받았다.

## 수행한 작업

- `IResettable` 인터페이스를 생성하여 매니저 초기화 공통 구조를 정의했다.
- 8개 매니저(DeckManager, StageManager, PawnManager, ItemManager, PlayerManager, LobbyManager, TurnManager, TileManager)에 `IResettable`을 구현하고 각각의 `ResetAll()` 메서드를 추가했다.
- `SceneController`에 `ReturnToTitle()` 메서드를 추가하여 OnExitScene → CloseAllView → ManagerRoot 하위 IResettable 자동 탐색 초기화 → TitleScene 로드 흐름을 구현했다.
- `BootScript.Awake()`에서 `DeckManager.Instance.InitTestData()` 호출을 제거했다.
- `LobbyScript.OnEnterScene()`에 보유 Pawn이 없을 때만 `InitTestData()`를 호출하는 조건부 로직을 추가했다.
- 기획서 3건을 갱신했다 (01_씬_흐름, 00_프로젝트_개요, 99_로드맵).

## 주요 결정 사항

- **ManagerRoot 자동 탐색 방식 채택**: `GetComponentsInChildren<IResettable>()`로 ManagerRoot 하위 매니저를 자동 발견하도록 구현했다. 명시적 호출 방식 대비 새 매니저 추가 시 IResettable만 구현하면 자동으로 초기화 대상에 포함되는 장점이 있다.
- **기존 OnExitScene 로직 유지**: StageScript.OnExitScene()의 기존 정리 로직(ClearAllTiles, ResetPawnStats, ResetForNextStage)은 Stage→Lobby 일반 전환에서 여전히 필요하므로 그대로 유지했다. ReturnToTitle 시 중복 실행되는 부분은 이미 빈 상태를 재정리하는 것이므로 부작용이 없다.
- **InitTestData 조건부 호출**: PawnManager.Pawns.Count == 0을 가드 조건으로 사용하여, 타이틀 복귀 후 ResetAll로 초기화된 상태에서 Lobby 재진입 시 자연스럽게 테스트 데이터가 재생성되도록 했다.
- **UIManager/DataManager/TextManager 제외**: UIManager는 CloseAllView()로 별도 처리되고, DataManager/TextManager는 정적 데이터이므로 IResettable 대상에서 제외했다.

## 참고 사항

- `ReturnToTitle()`을 호출하는 UI 버튼(OptionPopup 등)은 아직 연결되지 않았다. 타이틀 복귀가 필요한 UI에서 `SceneController.Instance.ReturnToTitle()`을 호출하면 된다.
- TurnManager.ResetAll()에서 `StopAllCoroutines()`를 호출하여 진행 중인 MonsterTurnCoroutine이 안전하게 중단된다.
- StageManager.ResetAll()에서 이벤트(onSelectedChanged, onBattleEnd)를 null로 설정하여 dangling 구독을 방어적으로 정리한다.
