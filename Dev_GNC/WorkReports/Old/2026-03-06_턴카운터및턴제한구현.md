# 작업 보고서: 턴 카운터 및 턴 제한 구현

**날짜**: 2026-03-06
**작업자**: Claude

## 요청 내용

StagePage의 `_textTurn`에 현재 턴 수를 표시하는 기능을 구현해달라는 요청. 턴 완전 종료는 Monster 행동이 끝난 이후로 취급하며, 턴 제한 초과 시 패배 처리 기반을 마련하는 것도 포함. Pawn 행동 차례 시작 시 턴이 갱신되고, StagePage는 이벤트를 구독해 UI를 업데이트하는 구조로 구현.

## 수행한 작업

- `TurnManager`에 `CurrentTurn`, `TurnLimit` 프로퍼티 및 `onTurnChanged` 이벤트 추가
- `TurnManager.SetTurnLimit()` 메서드 추가
- `TurnManager.StartPawnTurn()`에서 턴 카운터 증가 및 `onTurnChanged` 이벤트 발생 처리
- `TurnManager.MonsterTurnCoroutine()` 끝부분에 턴 제한 초과 판정 로직 삽입 (Monster 행동 완료 후 체크)
- `TileMapPreset`에 `turnLimit` 필드 추가 (0 = 제한 없음, Inspector에서 설정 가능)
- `StageManager.LoadStageFromPreset()`에서 preset의 `turnLimit`을 TurnManager에 전달
- `StageManager.OnTurnLimitExceeded()` 메서드 추가 — `onBattleEnd(false)` 발생시켜 패배 처리
- `StagePage.PreOpen()`에서 `onTurnChanged` 구독, `PreClose()`에서 구독 해제
- `StagePage.OnTurnChanged()` 핸들러 추가 — `_textTurn.text`를 현재 턴 수로 갱신

## 주요 결정 사항

- **턴 카운터 증가 타이밍을 `StartPawnTurn()` 진입부로 결정**: "Pawn의 행동 차례에 턴이 갱신"이라는 요구사항에 따라 Pawn 턴 시작 시 즉시 증가. `StartGame()`도 `StartPawnTurn(false)`를 호출하므로 1턴부터 자연스럽게 시작됨.
- **턴 제한 초과 판정을 Monster 행동 완료 후로 결정**: "5번째 턴의 유저 행동이 끝난 이후 몬스터 행동까지 모두 완료한 후 초과로 취급"이라는 요구사항을 반영하여 `MonsterTurnCoroutine()` 루프 종료 직후, `StartPawnTurn()` 호출 직전에 체크.
- **패배 트리거를 `StageManager.OnTurnLimitExceeded()`로 분리**: TurnManager가 `onBattleEnd` 이벤트를 직접 invoke할 수 없으므로(event는 선언 클래스 내에서만 invoke 가능), StageManager에 별도 메서드를 추가하여 호출하는 방식 채택.
- **StagePage 초기 텍스트는 이벤트로 처리**: `StageScript.OnEnterScene()`에서 StagePage가 먼저 열려 이벤트를 구독한 후 `TurnManager.StartGame()`이 호출되므로, 별도 초기화 코드 없이 이벤트로 자연스럽게 초기값(1) 표시.

## 참고 사항

- 턴 제한은 `TileMapPreset` Inspector의 `Turn Limit` 필드로 설정. `0`이면 무제한.
- 현재 `LoadStageDefault()`(preset 없이 기본 스테이지 로드)는 `SetTurnLimit()`을 호출하지 않으므로 기본값 0(무제한)으로 동작함.
- 이후 레벨 데이터(미션 타입, 난이도)에 따라 `turnLimit`을 활용할 예정으로, 현재는 패배 처리 기반만 마련된 상태.
