# 작업 보고서: LobbyUI 페이지 구성

**날짜**: 2026-03-06
**작업자**: Claude

## 요청 내용

ShopPage, DeckSettingPage, RecruitPage를 UI 페이지로 사용할 수 있도록 기초 구성을 요청하였으며, 이후 각 페이지의 닫기 버튼이 LobbyPage로 돌아오도록 수정을 요청하였다.

## 수행한 작업

- `ShopPage`, `DeckSettingPage`, `RecruitPage`를 `MonoBehaviour`에서 `PageView`로 변경
- 각 페이지에 `[AssetPath]` 어트리뷰트 추가 (`Prefabs/UI/{Name}/{Name}` 경로)
- `[Linker("Root/Button_Close")]` 닫기 버튼 링커 및 `PreOpen()` 리스너 등록 구성
- `OnClickClose()`를 `UIManager.Instance.CloseView(this)`로 구현하여 LobbyPage로 복귀
- `LobbyPage.PreOpen()`에서 누락된 `_buttonRecruit` 리스너 연결 추가
- `LobbyPage`의 빈 이벤트 메서드(`OnClickDeckSetting`, `OnClickShop`, `OnClickRecruit`)에 `UIManager.Instance.OpenView<T>()` 호출 완성

## 수정된 파일

| 파일 경로 | 변경 유형 | 변경 내용 요약 |
|-----------|-----------|----------------|
| `Assets/Scripts/UI/Shop/ShopPage.cs` | 수정 | PageView 상속, AssetPath/Linker/PreOpen/OnClickClose 구성 |
| `Assets/Scripts/UI/DeckSetting/DeckSettingPage.cs` | 수정 | PageView 상속, AssetPath/Linker/PreOpen/OnClickClose 구성 |
| `Assets/Scripts/UI/Recruit/RecruitPage.cs` | 수정 | PageView 상속, AssetPath/Linker/PreOpen/OnClickClose 구성 |
| `Assets/Scripts/UI/Lobby/LobbyPage.cs` | 수정 | _buttonRecruit 리스너 연결, 빈 이벤트 메서드에 OpenView 호출 완성 |

## 주요 결정 사항

- **CloseView(this) 사용**: `UIManager`에 `CloseView<T>()` 제네릭 메서드가 없고 `CloseView(ViewCore view)`만 존재하므로 `this`를 인자로 전달하는 방식 채택
- **LobbyPage 복귀 방식**: `OpenView`는 기존 페이지를 파괴하지 않고 최상단에 새 페이지를 쌓는 구조이므로, 현재 페이지만 닫으면 LobbyPage가 자연스럽게 노출됨. 별도의 LobbyPage 재오픈 불필요

## 참고 사항

각 페이지 프리팹을 Unity에서 생성 후 아래 경로에 배치해야 정상 동작한다.
- `Resources/Prefabs/UI/Shop/ShopPage`
- `Resources/Prefabs/UI/DeckSetting/DeckSettingPage`
- `Resources/Prefabs/UI/Recruit/RecruitPage`

프리팹 내 `Root/Button_Close` GameObject가 존재해야 Linker가 정상 연결된다.
