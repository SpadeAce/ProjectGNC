# 작업 보고서: HUD_Actor 구현

**날짜**: 2026-03-04
**작업자**: Claude

## 요청 내용

Stage 씬에서 Actor(Pawn, Monster) 머리 위에 HP, Shield, Armor, ActingPower를 표시하는 World Space UI인 HUD_Actor를 구현해달라는 요청. 공격/힐/쉴드 등 스탯 변경 시 즉시 업데이트되고, Actor 이동 및 카메라 시점 변경 시 항상 카메라를 바라보도록(빌보드) 처리해야 함.

## 수행한 작업

- `DPawn.cs`에 `onStatsChanged` 이벤트를 추가하고 `TakeDamage()`, `Heal()`, `AddShield()` 호출 시 이벤트 발화
- `DMonster.cs`에 동일한 `onStatsChanged` 이벤트 패턴 적용
- `Actor.cs`에 `[SerializeField] private HUD_Actor _hudActor` 필드 추가 및 `Init(DPawn)` / `Init(DMonster)` 에서 HUD 초기화 호출
- `HUD_Actor.cs`를 전면 구현: Init(), Refresh(), LateUpdate() 빌보드, OnDestroy() 이벤트 해제

## 수정된 파일

|-----------|-----------|----------------|

## 주요 결정 사항

- **World Space Canvas를 Actor 프리팹 자식으로 배치**: Actor 이동 시 HUD가 자동으로 따라가므로 별도 위치 추적 로직 불필요
- **빌보드를 LateUpdate에서 처리**: `transform.rotation = Camera.main.transform.rotation` 방식으로 카메라 회전에 즉시 반응
- **onStatsChanged 이벤트 구독/해제**: HUD_Actor의 Init() 호출 시 이전 구독 해제 후 재구독, OnDestroy()에서 반드시 해제하여 메모리 누수 방지
- **ActingPower는 DPawn 전용**: DMonster Init 시 `_goActingPower` 오브젝트를 비활성화

## 참고 사항

에디터에서 추가로 해야 할 프리팹 작업:
1. 각 Actor 프리팹(`Pawn_Soldier`, `Monster_Slime` 등) 하위에 `HUD_Actor` 빈 오브젝트 생성 (localPosition: 0, 1.8, 0)
2. HUD_Actor 하위에 World Space Canvas (Scale: 0.01) 추가
3. Canvas 하위에 HP_Fill(Image), Shield_Fill(Image), Text_Armor, AP_Group/Text_AP 구성
4. Actor 프리팹 Inspector에서 `_hudActor` 슬롯에 HUD_Actor 컴포넌트 연결, HUD_Actor Inspector에서 각 UI 요소 슬롯 연결
