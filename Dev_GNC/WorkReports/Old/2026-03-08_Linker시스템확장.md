# 작업 보고서: Linker 시스템 확장

**날짜**: 2026-03-08
**작업자**: Claude

## 요청 내용

기존 `[Linker(path, type)]` 어트리뷰트를 세 가지 방향으로 확장 요청.
① `type` 명시 제거 — LOCAL 우선 탐색 후 없으면 GLOBAL 자동 폴백.
② `params string[] paths`로 변경 — 경로를 여러 개 지정 가능.
③ `List<T>` 필드 지원 — 경로 수만큼 요소를 수집해 리스트로 대입 (단일 필드는 첫 번째 경로만 사용).

## 수행한 작업

- `LinkerAttribute.cs` 재작성 — `LINKER_TYPE` enum 및 `type` 필드 제거, 생성자를 `params string[] paths`로 변경
- `MonoBehaviourExInspector.cs` 재작성 — `ProcessUILinker()` 전면 개편
  - `List<T>` 필드 감지 (`IsGenericType` + `GetGenericTypeDefinition() == typeof(List<>)`)
  - List 필드: 모든 경로를 순회하며 요소를 수집해 리스트 생성 후 대입
  - 단일 필드: 첫 번째 경로만 사용하는 기존 방식 유지
  - `FindTarget()` 헬퍼 추출 — LOCAL(`transform.Find`) 우선, 실패 시 GLOBAL(`GameObject.Find`) 폴백
  - `GetValue()` 헬퍼 추출 — GameObject / Component 분기 처리
- Unity null 경고 수정 — `?.` 연산자 대신 명시적 null 체크로 교체 (`go != null ? go.transform : null`)

|-----------|-----------|----------------|

## 주요 결정 사항

- **`params string[]` 사용으로 하위 호환 유지**: 기존 `[Linker("Root/CardGroup")]` 형태는 단일 문자열이므로 `params` 덕분에 수정 없이 그대로 동작. 기존 사용처 전체 변경 불필요.
- **탐색 순서를 LOCAL → GLOBAL 자동 폴백으로 통일**: 명시적 `LINKER_TYPE` 지정을 제거하되, 프리팹 내부 경로(LOCAL)를 우선해 의도치 않은 씬 오브젝트 참조를 방지. 프로젝트 전체에서 `LINKER_TYPE.GLOBAL` 명시 사용 없음 — 제거 영향 없음.
- **Unity null 전파 연산자(`?.`) 미사용**: Unity는 `==` 연산자를 오버라이드하지만 `?.`는 오버라이드하지 않아 소멸된 오브젝트를 null로 인식하지 못하는 문제가 있어 명시적 null 체크로 교체.

## 참고 사항

- `List<T>` 필드는 `[SerializeField]`와 함께 사용 시 Inspector에서 별도로 표시되므로, "Set Linker" 버튼을 클릭한 후 Prefab에 Apply해야 값이 저장됨.
- 경로 중 일부가 존재하지 않아도 나머지는 정상 수집됨 (null 건너뜀).
