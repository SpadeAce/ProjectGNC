# 작업 보고서: Pawn 개인 카드 핸드 구현

**날짜**: 2026-03-06
**작업자**: Claude

## 요청 내용

각 Pawn이 개인 카드 핸드를 보유하고 매 턴 1장씩 드로우하도록 구현 요청.
게임 시작 시 3장으로 시작하며 최대 5장 보유 가능. Pawn 선택 시 StagePage의 `_goPawnCardGroup`에 해당 Pawn의 카드가 표시되고 드래그로 사용 가능. 현재는 글로벌 카드 풀을 공유 사용.

## 수행한 작업

- `DPawn.cs`에 `PawnHandCards`, `OnPawnHandChanged` 이벤트, `AddPawnHandCard`/`RemovePawnHandCard` 메서드 추가 (기존 미사용 `pawnCardList` 필드 교체)
- `DeckManager.cs`에 Pawn 핸드 관련 상수 3개(`InitialPawnHandSize=3`, `MaxPawnHandSize=5`, `PawnDrawPerTurn=1`) 및 `InitPawnHand`, `DrawPawnCards`, `UsePawnCard` 메서드 추가
- `StageScript.cs` Stage 진입 시 배치된 Pawn마다 `InitPawnHand` 호출하여 초기 핸드 세팅
- `TurnManager.cs` `StartPawnTurn`에서 매 턴 각 Pawn에게 `DrawPawnCards` 호출 추가
- `StageManager.cs` `UseCardSelf`/`TryUseCard`에 `DPawn cardOwner = null` 파라미터 추가, Pawn 카드 사용 시 `UsePawnCard` 분기 처리
- `StagePage.cs` `SetPawnCardList()` 메서드 구현, `_subscribedPawn` 필드로 이벤트 구독 관리, `StartCardDrag` 헬퍼 추출로 글로벌/Pawn 드래그 로직 통합, `OnPawnCardDragBegin` 핸들러 추가
- 기획서 `03_카드_시스템.md` 갱신 (DeckManager 구조/상수/메서드 표 업데이트)
- 기획서 `10_Pawn_카드_핸드_시스템.md` 신규 작성

## 주요 결정 사항

- **글로벌 풀 공유**: Pawn 전용 드로우 풀이 아직 없으므로 `DeckManager._cardPool`을 임시 공유. 추후 `DrawPawnCards` 내부만 수정하면 전용 풀로 교체 가능하도록 설계.
- **cardOwner 파라미터**: `StageManager.UseCardSelf`/`TryUseCard`에 `DPawn cardOwner = null` 선택적 파라미터 추가. null이면 글로벌 핸드, non-null이면 Pawn 핸드에서 제거. 기존 호출부 무변경으로 하위 호환 유지.
- **StartCardDrag 헬퍼**: `OnCardDragBegin`과 `OnPawnCardDragBegin`의 중복 로직을 `StartCardDrag(icon, eventData, DPawn owner)` 헬퍼로 추출. `_dragCardOwner` 필드로 DragEnd 시 분기.
- **구독 관리 방식**: `_subscribedPawn` 필드로 현재 구독 중인 Pawn을 추적. 선택 변경 시 이전 Pawn의 `OnPawnHandChanged`/`onStatsChanged` 구독을 해제 후 새 Pawn 구독. `onStatsChanged → SetCardList` 연결로 Pawn 카드 사용 시 AP 변경에 따른 글로벌 핸드 disabled 상태도 갱신.
