# 작업 보고서: 명중율 시스템 구현

**날짜**: 2026-03-18
**작업자**: Claude

## 요청 내용

Pawn의 공격에 명중율(Accuracy) 시스템을 구현한다. PawnData의 Accuracy 컬럼(만분율, 10000=100%)을 기반으로 Enemy 타겟 공격 시 명중 판정을 수행하고, 빗나가면 Evade 애니메이션과 MISS 텍스트를 표시한다. 카드 드래그 중 Enemy 위 호버 시 명중율 퍼센트를 UI에 표시한다.

## 수행한 작업

- PawnStats에 Accuracy 프로퍼티를 추가하고 Recalculate()에서 PawnData 기본값 + 장비 보너스 + 성장 보너스를 합연산하도록 구현
- DPawn에 Accuracy 런타임 프로퍼티를 추가하고 UpdateStatus()에서 Stats.Accuracy를 반영
- FloatingTextType에 Miss 열거값을 추가하고 회색(Color.gray) 색상 매핑
- Actor에 Evade() 메서드를 추가하여 "Evade" 애니메이터 스테이트를 재생 (ReceiveHit과 동일 패턴)
- Actor의 OnFloatingText에서 Miss 타입을 "MISS" 텍스트로 포맷팅
- StageManager의 ApplyCardEffectsCoroutine()에 카드 당 1회 명중 판정 로직을 추가 (Enemy 타겟 + caster=DPawn + target=DMonster 조건)
- Miss 시 Damage 미적용 + Evade 애니메이션 재생, Debuff도 스킵하도록 처리
- DMonster에 NotifyMiss() 메서드를 추가하여 event 외부 호출 문제(CS0070) 해결
- StagePage의 OnCardDragging()에서 Enemy 호버 시 명중율 퍼센트 텍스트를 표시하고, 비Enemy/드래그 종료 시 숨기도록 구현
- 전투 시스템 기획서(02_전투_시스템.md)에 명중율 시스템 섹션 추가
- 로드맵(99_로드맵.md)에 명중율 시스템 완료 항목 및 Ammo 리네임 완료 표기

## 주요 결정 사항

- **카드 당 1회 명중 판정**: 카드의 여러 효과(Damage + Debuff 등)에 대해 개별 판정이 아닌 1회 판정으로 통일. 이유: 하나의 공격 행위에서 데미지는 빗나가고 디버프만 적중하는 것은 부자연스러움
- **Enemy 타겟 전용**: Ground, Ally, Self 타겟 카드는 명중율 판정 없이 무조건 적용. 이유: 아군 힐/버프나 지면 효과에 명중 실패는 게임성에 부적합
- **DMonster.NotifyMiss() 래퍼 메서드 추가**: C# event는 선언 클래스 외부에서 Invoke 불가(CS0070). StageManager에서 호출할 수 있도록 DMonster 내부에 래퍼 메서드를 추가하여 해결
- **Miss 시 Heal/Shield/Buff는 정상 적용**: Damage와 Debuff만 스킵하여 복합 효과 카드에서도 유용한 부분은 적용되도록 설계

## 참고 사항

- 몬스터의 Animator Controller에 "Evade" 스테이트가 필요함. 미설정 시 애니메이션 미재생 (기능은 정상 동작)
- 현재 몬스터→Pawn 공격에는 명중율 미적용 (향후 Pawn 회피율 시스템으로 확장 가능, 로드맵에 기록됨)
- 명중율 표시는 정수 퍼센트(acc/100)로 표시하여 소수점 미표시 (예: 8500 → 85%)
