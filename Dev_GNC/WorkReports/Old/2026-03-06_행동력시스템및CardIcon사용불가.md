# 작업 보고서: 행동력 시스템 및 CardIcon 사용불가

**날짜**: 2026-03-06
**작업자**: Claude

## 요청 내용

로드맵에 정의된 행동력(ActingPower) 시스템을 구현하고, CardIcon에 사용불가 상태를 전환하는 기능을 추가하였다. 행동력 부족 시 카드 사용을 막고, 턴 시작 시 행동력을 회복하며, 카드 아이콘에 disabled 상태를 반영하는 것이 목표였다.

## 수행한 작업

- `DPawn`에 `HasEnoughActingPower()`, `ConsumeActingPower()`, `RestoreActingPower()` 메서드 추가
- `StageManager.UseCardSelf()`에 행동력 검증 및 차감 로직 추가
- `StageManager.TryUseCard()`에 행동력 검증(사거리 검증 직후) 및 각 분기별 차감 로직 추가
- `TurnManager.StartPawnTurn()`에서 모든 UserActor(Pawn)의 행동력을 턴 시작 시 전량 회복하도록 추가
- `CardIcon`에 `_isDisabled` 필드 및 `SetDisabled(bool)` 메서드 추가, 드래그 이벤트 핸들러에 disabled 가드 적용
- `StagePage.SetCardList()`에서 선택된 Pawn의 ActingPower를 기준으로 각 카드 아이콘에 `SetDisabled` 연동

|-----------|-----------|----------------|

| `Assets/Scripts/UI/Stage/StagePage.cs` | 수정 | SetCardList에서 선택된 Pawn ActingPower 기준 SetDisabled 연동 |

## 주요 결정 사항

- **행동력 검증 위치**: `StageManager.TryUseCard()`에서 사거리 검증 직후, 타겟 검증 전에 삽입 — 불필요한 타겟 탐색 없이 빠르게 실패 처리
- **차감 타이밍**: `DeckManager.UseCard()` (핸드 제거) 이후에 `ConsumeActingPower()` 호출 — 카드 소비와 스탯 변경 이벤트를 분리
- **비주얼 처리 분리**: `SetDisabled(bool)` 호출만 제공하고 시각적 표현은 사용자가 직접 구현 — 비주얼 로직을 CardIcon 내부에 강제하지 않음
- **disabled 갱신 시점**: `onHandChanged` 이벤트 → `SetCardList()` 흐름을 재활용. 행동력 차감은 카드 사용과 동시에 발생하므로 별도 이벤트 구독 불필요. 턴 시작 시 `StartTurnDraw()`가 `onHandChanged`를 발생시키므로 자연스럽게 갱신됨
- **Pawn 없을 때**: `SelectedObject`가 DPawn이 아닌 경우 disabled 처리 없음 (선택 없이는 카드 드래그 자체가 OnCardDragBegin에서 차단됨)

## 참고 사항

- CardIcon의 비주얼 처리(색상 변경, 반투명 등)는 사용자가 프리팹에서 직접 구현 예정
- `DMonster`는 ActingPower 개념이 없으므로 `caster.Data is DPawn` 체크로 안전하게 스킵됨
- 로드맵의 `RestoreAction` 카드 효과(행동력 회복)는 별도 구현 필요 — `DPawn.RestoreActingPower()`를 활용하면 됨
