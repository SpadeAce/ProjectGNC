# 작업 보고서: 전투 타이밍 및 입력 잠금

**날짜**: 2026-03-18
**작업자**: Claude

## 요청 내용

전투 중 공격 애니메이션이 끝난 후에야 데미지가 적용되어 싱크가 맞지 않는 문제와, 공격/피격 애니메이션 진행 중에도 다른 카드를 사용하거나 캐릭터를 이동시킬 수 있는 입력 잠금 부재 문제를 수정하도록 요청받았다. 추가로 플로팅 텍스트 크기가 3배 정도 크게 표시되는 문제와, 스테이지 전환 후 Actor 파괴 시 onFloatingText 이벤트 구독이 남아있어 MissingReferenceException이 발생하는 버그도 수정했다.

## 수행한 작업

- `FloatingText.cs`에 `BaseScale = 0.33f` 상수를 추가하고, `Show()`와 `PopInPlace()`에서 스케일 적용 시 BaseScale을 곱하도록 수정하여 텍스트 크기를 1/3로 축소했다.
- `Actor.cs`에 `OnDestroy()`를 추가하여 Actor 파괴 시 DPawn/DMonster의 `onFloatingText` 이벤트 구독을 해제하도록 수정했다.
- `StageManager.cs`에 `_isBusy` / `IsBusy` 플래그를 추가하고, `ApplyCardEffects`를 코루틴(`ApplyCardEffectsCoroutine`)으로 변환하여 카드 사용 시 애니메이션 완료까지 입력을 잠그도록 구현했다.
- `StageManager.cs`의 Damage 효과 처리에서 공격 모션 시작 후 0.3초 고정 딜레이에 데미지를 적용하고, 피격 모션을 병렬로 시작하도록 타이밍을 변경했다.
- `TurnManager.cs`의 `ExecuteAttack`도 동일한 0.3초 딜레이 타이밍 방식으로 변경하여 몬스터 공격 시에도 모션 중간에 데미지가 적용되도록 수정했다.
- `StageScript.cs`의 `Update()`에 `StageManager.Instance.IsBusy` 가드를 추가하여 애니메이션 진행 중 타일 클릭을 차단했다.
- `StagePage.cs`의 `StartCardDrag()`에 `StageManager.Instance.IsBusy` 가드를 추가하여 애니메이션 진행 중 카드 드래그를 차단했다.

## 주요 결정 사항

- **고정 딜레이 방식 채택 (0.3초)**: AnimationEvent 대신 `WaitForSeconds(0.3f)`로 타격 타이밍을 잡았다. 현재 프로젝트에 AnimationEvent 사용 패턴이 없고, 모든 액터가 유사한 공격 모션을 사용하므로 고정값으로 충분하다. 향후 애니메이션별 타이밍 차이가 필요하면 AnimationEvent로 전환 가능하다.
- **StageManager.IsBusy 중앙 집중 방식**: 개별 UI 컴포넌트마다 잠금 플래그를 두지 않고, StageManager에 단일 `IsBusy` 플래그를 두어 StageScript와 StagePage 양쪽에서 참조하도록 했다.
- **ApplyCardEffects 코루틴 변환**: 기존 콜백 중첩(3중 콜백) 구조를 코루틴의 순차 yield로 변환하여 가독성과 흐름 제어를 개선했다.

## 참고 사항

- 0.3초 딜레이 값은 실제 공격 애니메이션의 타격 프레임에 따라 조정이 필요할 수 있다.
- 몬스터 턴은 기존에도 코루틴으로 순차 처리되어 플레이어 입력이 불가능하므로, IsBusy 가드가 별도로 필요하지 않다.
