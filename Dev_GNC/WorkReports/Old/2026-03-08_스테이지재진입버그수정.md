# 작업 보고서: 스테이지 재진입 버그 수정

**날짜**: 2026-03-08
**작업자**: Claude

## 요청 내용

전투 완료 후 로비에서 스테이지에 재진입할 때 `ArgumentException: An item with the same key has already been added. Key: 0` 에러가 발생하는 문제 수정 요청.
또한 스테이지 재진입 시 이전 상태가 잔류하지 않는지 전체적인 점검 요청.

## 수행한 작업

- `StageManager` 전체 점검 — `_dicUserActor`, `_dicEnemyActor` 딕셔너리가 재진입 시 Clear되지 않아 중복 키 예외 발생 확인
- `ResetStageState()` 헬퍼 메서드 추출 — 딕셔너리 2개, HashSet 3개(`_reachableTiles`, `_cardRangeTiles`, `_radiusPreviewTiles`), 선택 상태(`Deselect()`) 일괄 초기화
- `LoadStageFromPreset()`, `LoadStageDefault()` 각각 `ResetStageState()` 호출로 교체
- `TurnManager.StartGame()` 점검 — `CurrentTurn`이 누적되는 문제 발견, `CurrentTurn = 0` 리셋 추가
- `DeckManager.InitStage()` 점검 — 각 Pawn의 `PawnHandCards`가 초기화되지 않아 이전 스테이지 핸드 잔류 문제 발견, `pawn.PawnHandCards.Clear()` 추가
- 로드맵(`99_로드맵.md`) 정리 — 완료된 항목 제거, 향후 작업(`08_자원_시스템.md`, `09_상점_시스템.md`) 기반으로 신규 일감 추가

## 주요 결정 사항

- **Gold는 리셋하지 않음**: Gold는 로비 메타 자원으로 스테이지 간 유지가 올바른 설계. `DeckManager.InitStage()`에서 초기화 대상에서 제외.
- **`ResetStageState()` 메서드 추출**: `LoadStageFromPreset`과 `LoadStageDefault` 두 경로에서 동일한 초기화가 필요하므로 중복 제거를 위해 별도 메서드로 추출.

## 참고 사항

- `TileManager`는 `GenerateTileMap`/`GenerateTileMapFromPreset` 호출 시 내부적으로 `ClearAllTiles()`를 통해 완전 초기화되므로 별도 수정 불필요.
- `TurnManager._remainingMovement`는 `StartPawnTurn()` 진입 시마다 `Clear()`되므로 문제 없음.
- Pawn의 버프/디버프 상태(`DPawn` 내 buff list)는 스테이지 간 초기화 여부 미확인 — 후속 점검 필요.
