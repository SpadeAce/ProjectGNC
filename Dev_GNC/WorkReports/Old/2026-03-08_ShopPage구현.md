# 작업 보고서: ShopPage 구현

**날짜**: 2026-03-08
**작업자**: Claude

## 요청 내용

ShopData의 Level을 기준으로 카드를 진열하고 구매 기능을 갖춘 ShopPage를 구현해달라는 요청.
상점 레벨업, 5×2 그리드 카드 진열, 구매 시 DeckManager 연동, Stage 진행 전까지 진열 유지 등 5가지 세부 요건 포함.
또한 이전 작업인 RecruitItem의 AssetPath 기반 로드 방식도 함께 수정.

## 수행한 작업

- `RecruitPage.RefreshItems()`에서 하드코딩된 `Resources.Load<GameObject>` 제거 → `AssetPath` Reflection 방식으로 교체
- `ShopTable`에 Load 시점에 레벨 기반 Dictionary(`_levelMap`)와 최대 레벨(`_maxLevel`)을 구축하는 최적화 적용
- `ShopTable`에 `GetByLevel(int level)`, `GetMaxLevel()` 메서드 추가
- `DeckManager`에 `AddCard(DCard card)` 퍼블릭 메서드 추가
- `ShopItem`을 `SetData(CardData)`로 교체하고 구매 로직(DCard 생성 → DeckManager 등록 → hide → 이벤트 발행) 구현
- `ShopPage` 전체 구현: 레벨 기반 카드 목록 생성, 5×2 그리드 배치, 레벨업 버튼, Stage 전까지 진열 유지
- `06_UI_시스템.md` 기획서 갱신 (ShopPage / RecruitPage 구현 상태 업데이트)
- `99_로드맵.md` 갱신 (상점 관련 항목 완료 처리)
- `09_상점_시스템.md` 신규 기획서 작성

## 주요 결정 사항

- **ShopTable 레벨 맵 선구축**: `GetByLevel()` 호출마다 전체 순회하는 대신 Load 시점에 `_levelMap`과 `_maxLevel`을 한 번에 구축하여 O(1) 조회로 최적화.
- **ShopItem.SetData 시그니처 변경**: 기존 `SetData(ShopData)`는 상점 전체 데이터를 개별 아이템에 전달하는 잘못된 설계였으므로 `SetData(CardData)`로 교체. ShopData → CardId 매핑은 ShopPage가 담당.
- **구매 후 hide 처리**: RecruitPage처럼 Destroy하지 않고 `SetActive(false)`로 숨김 처리 (요구사항 명시).
- **AssetPath Reflection 패턴 통일**: RecruitPage와 ShopPage 모두 `typeof(T).GetCustomAttribute<AssetPathAttribute>()?.Path` 방식으로 일관성 유지.

## 참고 사항

- ShopPage 프리팹에 `Root/Button_LevelUp` 경로로 LevelUp 버튼 추가 필요.
- ShopItem 카드 배치 간격(x: 220f, y: 300f)은 실제 프리팹 크기에 맞게 조정 필요.
- 골드 시스템 미구현으로 현재 모든 카드는 가격 표시만 하고 실제 골드 차감 없이 구매 가능.
- Stage 클리어 후 `ShopPage.Refresh()` 연동은 향후 작업.
