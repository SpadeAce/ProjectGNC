# 작업 보고서: Pawn 덱 편성 구현

**날짜**: 2026-03-08
**작업자**: Claude

## 요청 내용

DeckSettingPage에 Pawn/Card 탭 전환을 구현하고, Pawn 탭에서 보유 Pawn을 덱 슬롯에 편성하는 기능 구현 요청.
DeckManager는 "보유 전체 Pawn"과 "덱 편성 슬롯"을 분리해 관리하도록 수정.
영입한 Pawn은 보유 목록에만 포함하고, 별도 편성 과정을 거쳐 덱에 배치.

- `DeckManager` API 추가 — `AssignPawnToDeck`, `UnassignPawnFromDeck`, `FindEmptySlotIndex`, `GetDeckPawn`, `OpenSlotCount`

- `DeckPawnItem.cs` 구현 — `SetData(DPawn)`, 드래그 이벤트(`OnDragBegin/Dragging/DragEnd`), 우클릭 이벤트(`OnRightClick`)
- `DeckPawnSlot.cs` 구현 — `Init(slotIndex, isLocked)`, `SetData(DPawn)`, `Clear()`, 우클릭 이벤트
- `DeckSetting_Pawn.cs` 구현 — 슬롯 초기화, 보유 Pawn 목록 생성, 드래그 드롭/우클릭 편성 로직
- `DeckSettingPage.cs` 수정 — `OnOpened()` 추가 → `_pawn.OnOpened()` 호출
- `StageManager.cs` 수정 — `DeckPawns.ToList()` 변환으로 컴파일 에러 수정 (IEnumerable → List, `.Count` 프로퍼티 및 `[i]` 인덱서 사용 가능)

## 주요 결정 사항

- **_ownPawnList / _deckPawnSlots 분리**: 영입한 Pawn은 보유 목록에만 추가하고, 덱 편성은 별도 5슬롯 배열로 관리. Stage 진행 시 `DeckPawns`(null 제외)만 사용하므로 편성 안 된 Pawn은 전투에 참여하지 않음.
- **초기 열린 슬롯 2개**: `_openSlotCount = 2`. 잠긴 슬롯(index >= openSlotCount)은 `_goLock` 활성화 + `AssignPawnToDeck` 내부에서 배정 차단. 추후 `_openSlotCount` 값만 늘려 확장 가능.
- **DeckPawns IEnumerable 반환으로 StageManager 에러**: `Mathf.Min(a, b, deckPawns.Count)`에서 `.Count`가 LINQ 확장 메서드(메서드 그룹)로 인식되어 컴파일 에러 발생. StageManager에서 `.ToList()` 변환으로 해결. DeckManager 반환 타입은 IEnumerable 유지 (LINQ Where 결과를 매 호출마다 평가).
