# 작업 보고서: 액터 선택 전환 개선

**날짜**: 2026-03-05
**작업자**: Claude

## 요청 내용

선택된 캐릭터가 있는 상태에서 다른 캐릭터를 클릭하면 기존에는 선택 해제 후 재클릭이 필요했는데,
한 번의 클릭으로 선택이 즉시 전환되도록 변경 요청.
추가로 턴 종료 시 현재 선택 중인 캐릭터의 선택이 자동으로 해제되도록 요청.

## 수행한 작업

- `StageManager.SelectTile()` Case 2 로직 분석 — 이동 불가 타일 클릭 시 항상 `Deselect()` 후 `return`하여 Case 3이 실행되지 않는 구조 확인
- Case 2의 비이동 분기를 재구성하여 클릭한 타일에 액터가 있을 경우 Case 3으로 fall-through하도록 수정
- 이동 분기(`IsEmptyTile && canMove`)에 명시적 `return` 추가하여 이동 처리 후 정상 종료 보장
- `Deselect()` 접근 제한자를 `private` → `public`으로 변경
- `TurnManager.EndPawnTurn()` 첫 줄에 `StageManager.Instance.Deselect()` 호출 추가

## 수정된 파일

| 파일 경로 | 변경 유형 | 변경 내용 요약 |
|-----------|-----------|----------------|
| `Assets/Scripts/Logic/Manager/StageManager.cs` | 수정 | `SelectTile()` Case 2 fall-through 처리; `Deselect()` public으로 변경 |
| `Assets/Scripts/Logic/Manager/TurnManager.cs` | 수정 | `EndPawnTurn()` 첫 줄에 `StageManager.Instance.Deselect()` 추가 |

## 주요 결정 사항

- **fall-through 방식 채택**: Case 2에서 별도 선택 로직을 중복 작성하지 않고, 조건 미충족 시 Case 3으로 자연스럽게 이어지는 구조를 선택하여 코드 중복 최소화.
- **턴 종료 선택 해제는 TurnManager에서 처리**: StagePage(UI 레이어)에서 처리하면 버튼 외의 경로(미래의 자동 턴 종료 등)에서 누락될 수 있으므로, 턴 종료의 권한을 가진 TurnManager에서 호출.

## 참고 사항

- 기존 동작(빈 타일 클릭 시 해제, 이동 가능 타일 클릭 시 이동)은 그대로 유지됨.
- 몬스터 타일을 클릭해도 fall-through되어 선택 전환 시도가 일어나지만, Case 3의 `canShowRange` 조건(`Data is DPawn`)에 의해 이동 범위 표시는 되지 않음 — 의도된 동작.
