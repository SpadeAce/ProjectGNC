# 작업 보고서: 카드 드래그 타겟 오브젝트 전환

**날짜**: 2026-03-14
**작업자**: Claude

## 요청 내용

카드 드래그 시 취소 영역(CancelRadius=100px) 밖으로 나가면 카드 이미지 대신 타겟 유형에 맞는 오브젝트(_goEnemyTarget / _goAllyTarget)를 마우스 위치에 표시하도록 구현 요청. 취소 영역 안으로 돌아오면 다시 카드가 마우스에 붙는 원래 동작으로 복귀.

## 수행한 작업

- `StartCardDrag`에서 드래그 시작 시 두 타겟 오브젝트를 비활성화하는 초기화 코드 추가
- `OnCardDragging`을 재작성하여 취소 영역 여부에 따라 카드/타겟 오브젝트 전환 로직 구현
  - 취소 영역 밖 + Enemy/Ground 타겟 → 카드 숨기고 `_goEnemyTarget` 마우스 위치로 이동
  - 취소 영역 밖 + Ally 타겟 → 카드 숨기고 `_goAllyTarget` 마우스 위치로 이동
  - 취소 영역 안 → 카드 표시, 타겟 오브젝트 비활성화 (기존 동작)
- `OnCardDragEnd`에 드래그 종료 시 카드 `SetActive(true)` 복원 및 두 타겟 오브젝트 비활성화 추가

## 수정된 파일

|-----------|-----------|----------------|
| `Assets/Scripts/UI/Stage/StagePage.cs` | 수정 | StartCardDrag 초기화, OnCardDragging 전환 로직, OnCardDragEnd 정리 |

## 주요 결정 사항

- **카드 가시성 제어에 `SetActive` 사용**: 위치를 화면 밖으로 옮기는 대신 `SetActive(false)`로 카드를 숨겨 명확하게 처리
- **`icon.DragStartPosition` 재사용**: 취소 영역 판정에 `OnCardDragEnd`와 동일한 기준점(`icon.DragStartPosition`)을 사용하여 일관성 유지

## 참고 사항

- `_goEnemyTarget`, `_goAllyTarget`은 StagePage 프리팹의 `Root/Image_EnemyTarget`, `Root/Image_AllyTarget`에 Linker로 연결된 오브젝트
- EnemyTarget과 AllyTarget은 동시에 활성화되지 않는 구조 (상호 배타적)
- Self 타겟 카드는 타겟 오브젝트 전환 대상에서 제외 (기존 수직 드래그 판정 유지)
