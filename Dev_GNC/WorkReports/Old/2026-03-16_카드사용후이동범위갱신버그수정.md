# 작업 보고서: 카드 사용 후 이동 범위 갱신 버그 수정

**날짜**: 2026-03-16
**작업자**: Claude

## 요청 내용

Pawn을 선택한 상태에서 카드를 사용하면 이동 가능 타일이 갱신되지 않는 버그 수정 요청.
길을 막는 Monster를 처치하거나 BuffMovement 카드를 사용해도 이동 범위가 변하지 않아 선택 해제 후 재선택해야 하는 문제.

## 수행한 작업

- 원인 분석: `StagePage.OnCardDragEnd()`에서 `RestoreMovementRange()`가 카드 사용 **전**에 호출되고, 사용 **후** 갱신이 없음을 확인
- 원인 분석: `BuffMovement` 적용 시 `TurnManager._remainingMovement`가 보정되지 않아 잔여 이동력이 이전 값 그대로인 문제 확인
- 원인 분석: Damage → Die가 비동기(애니메이션 콜백)라 몬스터 제거 후에도 BFS 재계산이 없는 문제 확인
- `TurnManager.cs`에 `AdjustRemainingMovement(Actor, int delta)` 메서드 추가
- `StageManager.cs`에 `RefreshMovementRange()` 메서드 추가
- `ApplyCardEffects` 내 Damage → Die 콜백에 `RefreshMovementRange()` 호출 추가
- `ApplyCardEffects` 내 BuffMovement 처리에 `AdjustRemainingMovement` + `RefreshMovementRange()` 호출 추가
- `ApplyCardEffects` 내 DebuffMovement 처리에 `AdjustRemainingMovement` + `RefreshMovementRange()` 호출 추가

## 수정된 파일

| 파일 경로 | 변경 유형 | 변경 내용 요약 |
|-----------|-----------|----------------|
| `Assets/Scripts/Logic/Manager/TurnManager.cs` | 수정 | `AdjustRemainingMovement()` 메서드 추가 |
| `Assets/Scripts/Logic/Manager/StageManager.cs` | 수정 | `RefreshMovementRange()` 추가, `ApplyCardEffects` 내 3곳에 갱신 호출 추가 |

## 주요 결정 사항

- **갱신 위치를 ApplyCardEffects 내부에 배치**: 카드 사용 후 일괄 갱신 대신, 각 효과(Damage/Buff/Debuff) 처리 직후 즉시 갱신하는 방식 선택. 비동기 콜백(Die)과 동기 효과(Buff)를 각각 적절한 시점에 처리하기 위함.
- **AdjustRemainingMovement는 이동한 Actor만 보정**: `_remainingMovement`에 없는 Actor(아직 이동 전)는 `GetEffectiveMovement()`가 `pawn.Movement`를 직접 읽으므로 별도 보정 불필요.
