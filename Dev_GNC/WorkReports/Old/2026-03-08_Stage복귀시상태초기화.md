# 작업 보고서: Stage 복귀 시 상태 초기화

**날짜**: 2026-03-08
**작업자**: Claude

## 요청 내용

Stage 진행 후 로비로 복귀했을 때 상점/영입 목록이 갱신되지 않는 문제 수정 요청.
추가로 Stage 중 소모된 Pawn HP, Shield, 적용된 버프/디버프도 복귀 시 초기화되어야 한다는 요청.

## 수행한 작업

- `LobbyManager`에 `ResetForNextStage()` 추가 — `_shopCards`, `_recruitPawns`를 `null`로 초기화하여 다음 로비 진입 시 목록을 새로 생성하도록 함
- `DPawn`에 `ResetStats()` 추가 — `_buffs.Clear()` 후 `UpdateStatus()` 호출하여 버프/디버프 제거 및 HP·Shield·Armor·Ammo 등 전체 스탯 기본값 복원
- `DeckManager`에 `ResetPawnStats()` 추가 — 덱의 모든 Pawn에 `ResetStats()` 일괄 호출
- `StageScript.OnExitScene()`에서 `DeckManager.Instance.ResetPawnStats()` 및 `LobbyManager.Instance.ResetForNextStage()` 호출 연결

## 주요 결정 사항

- **상점/영입 목록 갱신 트리거를 `StageScript.OnExitScene()`에 배치**: Stage 종료 시점이 명확한 갱신 기준이며, ShopPage/RecruitPage의 `Refresh()` 직접 호출보다 페이지 오픈 여부와 무관하게 안정적으로 동작함. 다음 ShopPage/RecruitPage 열기 시 `ShopCards == null` 조건으로 자동 생성됨.
- **Pawn 스탯 리셋을 `DPawn.ResetStats()`로 캡슐화**: 버프 역산(ApplyStatDelta 역방향) 없이 `_buffs.Clear()` 후 `UpdateStatus()` 재호출로 단순화. 기본값이 PawnData에 있으므로 언제나 정확한 초기 스탯으로 복원 가능.

## 참고 사항

- Stage 종료 흐름: `BattleResultPopup` Continue 버튼 → LobbyScene 이동 → `LobbyScript.OnEnterScene()` → `LobbyPage` 오픈. `StageScript.OnExitScene()`은 씬 이동 직전에 호출되므로 로비 복귀 전 초기화가 보장됨.
- 추후 로그라이크 방식(HP가 영구 감소하는 등)으로 설계가 변경될 경우 `ResetStats()` 대신 부분 복원 메서드를 추가하고, `ResetPawnStats()`의 호출 여부를 설계에 맞게 조정해야 함.
