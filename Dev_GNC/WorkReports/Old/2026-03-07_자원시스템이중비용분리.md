# 작업 보고서: 자원 시스템 이중 비용 분리

**날짜**: 2026-03-07
**작업자**: Claude

## 요청 내용

CardData의 `cost` 필드가 `energyCost`(공용 자원)와 `ammoCost`(Pawn 개별 자원)로 분리됨에 따라, 관련 로직 전체를 수정했다. 공용 행동력(SharedActingPower)을 TurnManager에 추가하고, Pawn의 개별 자원을 Ammo로 명칭 변경하며 매 턴 자동 회복을 제거했다. RestoreAction은 공용 자원을 회복하도록, Reload는 Pawn 탄약을 기본값으로 전체 회복하도록 변경/신규 구현했다.

## 수행한 작업

- `DPawn.ActingPower`를 `Ammo`로 rename (프로퍼티 및 관련 메서드 전체)
- Pawn의 매 턴 자동 Ammo 회복 로직 제거 (`TurnManager.StartPawnTurn`에서 `RestoreActingPower()` 호출 삭제)
- `TurnManager`에 `SharedActingPower` 추가 (매 턴 5로 리셋, 카드 효과로 한도 초과 허용)
- `StageManager`에 `CanAffordCard` / `SpendCardCost` 헬퍼 추가 — EnergyCost는 공용 AP, AmmoCost는 Pawn Ammo에서 소모
- `RestoreAction` 효과를 공용 `SharedActingPower` 회복으로 변경
- `Reload` 효과 신규 구현 — 대상 Pawn의 Ammo를 `PawnData.ActingPower` 기본값으로 전체 회복 (value 무시)
- `StagePage`의 카드 비활성화 체크를 SharedActingPower + Pawn Ammo 두 자원 모두 확인하도록 수정
- `StagePage`에 `onSharedActingPowerChanged` 이벤트 구독 추가 (공용 AP 변경 시 카드 활성화 자동 갱신)
- `HUD_Actor`, `CardIcon`, `BattleCard`의 구식 참조(`ActingPower`, `card.Data.Cost`) 갱신
- 기획서 `08_자원_시스템.md` 신규 작성
- 기획서 `03_카드_시스템.md` 내용 업데이트 (CardData 필드, 효과 구현 상태, 자원 연동 섹션)

## 주요 결정 사항

- **SharedActingPower는 TurnManager에 배치**: 공용 자원은 게임 전체 상태이므로 Pawn이 아닌 TurnManager에서 관리. MonoSingleton 규칙상 SerializeField 없이 코드로만 초기화.
- **Ammo 매 턴 자동 회복 제거**: 탄약은 희소성이 있어야 하므로 Reload 카드를 직접 사용해야만 회복. 기본값 리셋 방식(`PawnData.ActingPower`)으로 통일하여 value 파라미터 무시.
- **Reload 효과의 value 무시**: 부분 회복이 아닌 전체 회복 방식을 채택. 데이터 테이블에서 value를 설정해도 적용되지 않으므로 기획 시 주의 필요.
- **카드 비활성화 조건 AND 처리**: EnergyCost와 AmmoCost 중 하나라도 부족하면 카드 비활성화. OR 조건이 아님에 유의.

## 참고 사항

- `PawnData.ActingPower` 필드명은 현재 Ammo 초기값으로 재활용 중. 추후 proto 파일에 별도 `Ammo` 필드 추가를 고려할 것.
- SharedActingPower의 UI 표시가 아직 없음 — HUD에 공용 AP 수치를 보여주는 UI 요소 추가 필요 (향후 작업).
- AmmoCost가 있는 카드의 카드 아이콘에 탄약 비용이 표시되지 않음 — 카드 UI에 AmmoCost 표시 추가 필요 (향후 작업).
