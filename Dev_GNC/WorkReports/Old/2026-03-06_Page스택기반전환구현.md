# 작업 보고서: Page 스택 기반 전환 구현

**날짜**: 2026-03-06
**작업자**: Claude

## 요청 내용

Page가 오픈된 상태에서 다른 Page가 열릴 경우, 위에 쌓는 형태가 아니라 기존 Page를 Hide(SetActive false)하고 전환하는 방식으로 수정. 새 Page가 닫히면 이전 Page가 다시 Show(SetActive true)됨. PageView에 `_dontHide` 옵션 추가로 특정 페이지는 새 페이지 오픈 시에도 숨겨지지 않도록 함.

## 수행한 작업

- `PageView`에 `[SerializeField] private bool _dontHide` 필드 및 `DontHide` 프로퍼티 추가
- `UIManager`에 `Stack<PageView> _pageStack` 필드 추가
- `UIManager.OpenView<T>()`: PageView 열 때 현재 top 페이지를 `DontHide` 여부에 따라 `SetActive(false)` 처리 후 스택에 Push
- `UIManager.CloseView()`: PageView 닫을 때 스택에서 Pop 후 이전 top 페이지를 `SetActive(true)`로 복원
- `UIManager.CloseAllView()`: `_pageStack.Clear()` 추가

## 수정된 파일

| 파일 경로 | 변경 유형 | 변경 내용 요약 |
|-----------|-----------|----------------|
| `Assets/Scripts/Logic/UI/PageView.cs` | 수정 | `_dontHide` 필드 및 `DontHide` 프로퍼티 추가 |
| `Assets/Scripts/Logic/Manager/UIManager.cs` | 수정 | `_pageStack` 추가, OpenView/CloseView/CloseAllView 스택 로직 적용 |

## 주요 결정 사항

- **Stack 자료구조 선택**: 페이지 내비게이션은 항상 LIFO 순서로 닫힌다는 가정 하에 Stack 사용. 중간 페이지 강제 닫기는 현재 요구사항 범위 밖으로 처리하지 않음
- **이미 열린 페이지 재요청 시**: 기존 인스턴스를 그대로 반환하고 스택/hide 로직 미적용 (이미 화면에 있는 상태이므로)
- **PopupView는 변경 없음**: 스택 관리는 PageView 전용으로만 적용

## 참고 사항

- `_dontHide`는 프리팹 기반 PageView에서만 Inspector로 설정 가능
- 페이지 전환 시 실제 Destroy가 아닌 SetActive(false)이므로 이전 페이지 상태가 유지됨
