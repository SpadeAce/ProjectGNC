# 작업 보고서: UseCard 효과 구현

**날짜**: 2026-03-04
**작업자**: Claude

## 요청 내용

카드 사용 시 CardEffectData 테이블을 읽어 Damage / Heal / Shield 효과를 적용하는 기능을 구현해달라는 요청.
Damage 효과의 경우 공격자의 "Attack01" 애니메이션 재생 후 피해를 적용하고, 피격 대상은 공격자를 바라보며 "GetHit" 애니메이션을 재생해야 한다.

## 수행한 작업

- `CardEffectTable`, `TileEntityTable`을 기존 테이블 패턴(PawnTable, MonsterTable 등)에 맞춰 구현
- `DataManager`에 `CardEffectTable`, `TileEntityTable` 프로퍼티 추가 및 `LoadAll()`에 로드 코드 추가
- `DPawn`에 `Heal(int amount)`, `AddShield(int amount)` 메서드 추가
- `DMonster`에 `Heal(int amount)`, `AddShield(int amount)` 메서드 추가
- `Actor`에 `ReceiveHit(Actor attacker)` 메서드 추가 — 공격자를 바라보고 "GetHit" 재생
- `StagePage.UseCard()`의 TODO를 실제 구현으로 교체 — `CardData.EffectId` / `EffectValue` 루프를 돌며 효과 적용

## 주요 결정 사항

- **EffectId / EffectValue 복수 지원**: `CardData.EffectId`와 `EffectValue`가 모두 `RepeatedField<int>`이므로 인덱스를 맞춰 루프 처리. 카드 하나에 여러 효과가 붙는 구조를 처음부터 지원.
- **Damage 시 애니메이션 순서**: `caster.PerformAttack()` 콜백(`onComplete`)에서 `TakeDamage()`와 `ReceiveHit()`을 동시에 호출. Attack01이 끝나는 시점에 피해와 피격 모션이 함께 발생.
- **`SA.DObject` 명시적 네임스페이스**: StagePage에서 `DObject`를 `SA.DObject`로 명시해 컴파일 오류 해결.
- **Unity null 전파 연산자 회피**: `targetActor?.Data` 대신 `targetActor != null ? targetActor.Data : null`로 작성 (Unity 오브젝트 null 전파 경고 대응).

## 참고 사항

- Heal 최대값은 `Data.Hp`(프로토버프 원본 스탯) 기준으로 캡 처리.
- 현재 `UseCard`는 Damage / Heal / Shield 3종만 처리하며, 나머지 EffectType(RestoreAction, Buff/Debuff 등)은 추후 확장 예정.
- `GetHit` 애니메이터 스테이트 이름이 실제 Animator Controller와 일치해야 정상 동작.
