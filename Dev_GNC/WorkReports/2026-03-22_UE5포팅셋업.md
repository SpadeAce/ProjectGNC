## 주요 결정 사항

- **MonoSingleton → Subsystem 매핑**: 영속 매니저 7개는 `UGameInstanceSubsystem`, 레벨 종속 매니저 3개는 `UWorldSubsystem`으로 분류. WorldSubsystem 자동 파괴로 IResettable 불필요
- **Variant 패턴 채택**: 기존 Variant_Strategy/TwinStick 구조에 맞춰 Variant_CardGame으로 추가
- **C++ 주도 개발**: 로직은 C++, UI 레이아웃/애니메이션만 Blueprint 사용
- **데이터 포맷 전환**: Protobuf .bytes → UDataTable (CSV/JSON). FTableRowBase 구조체로 수동 정의
- **포팅 순서**: 데이터 → 타일맵 → 전투 → UI → 씬흐름 → 에셋 (6단계)
