## 배경

`FStagePresetRow`의 `PresetPath` 필드가 `TArray<FString>`으로 선언되어, `StaticLoadObject`로 에셋을 로드하고 있었다. 이 방식은:
- 에디터에서 에셋 경로를 수동 입력해야 하고
- 경로 오타 시 런타임에서만 에러가 발생하며
- 에셋 참조 추적(Reference Viewer)이 불가능

Proto 엑셀의 FieldType에 `enum:EnumName` 패턴처럼 `asset:ClassName` 키워드를 추가하여, C++ 쪽에서 `TSoftObjectPtr<UClassName>`으로 매핑되도록 파이프라인을 확장했다.

## 구현 내용

### 1. Python 파이프라인 — `asset:` FieldType 지원

| 파일 | 변경 내용 |
|------|-----------|
| `Tools/proto_generator.py` | `_to_proto_type()`: `asset:X` → proto `string`으로 변환 |
| `Tools/validator.py` | `_is_valid_cell()`: `asset:` 타입을 문자열로 검증 |
| `Tools/validator.py` | `_validate_schema()`: `asset:` 접두사를 알 수 없는 타입 경고에서 제외 |
| `Tools/data_serializer.py` | `_coerce_scalar()`: `asset:` 타입을 `str()`로 처리 |

`json_exporter.py`는 변경 불필요 — 문자열 경로가 그대로 JSON에 출력되며, UE5 DataTable 임포트 시 `TSoftObjectPtr`에 자동 바인딩된다.

### 2. C++ 코드 변경

| 파일 | 변경 내용 |
|------|-----------|
| `CardGameRowTypes.h` | `UTileMapPreset` forward declaration 추가 |
| `CardGameRowTypes.h` | `TArray<FString>` → `TArray<TSoftObjectPtr<UTileMapPreset>>` |
| `CardStageGameMode.cpp` | `StaticLoadObject` → `TSoftObjectPtr::LoadSynchronous()` |

### 3. 가이드 문서 갱신

`WorkReports/Guide/DataTable_가이드.md`에 **스키마 FieldType 참조** 섹션 추가:
- 기본 타입 / 특수 타입(`enum:`, `asset:`) 매핑 테이블
- `asset:` 타입 사용법 (스키마, C++ 구조체, 로드 코드, 데이터 입력값 예시)
- `repeated` 필드 설명
