# Phase 4: 카드/덱 시스템 구현

> **날짜**: 2026-03-24
> **작업 유형**: 기능 구현 (Unity→UE5 포팅)
> **Phase**: Phase 4 — 카드/덱 시스템

---

## 작업 요약

Unity C# 카드게임의 DeckManager, PlayerManager, PawnManager, ItemManager를 UE5 C++ GameInstanceSubsystem 4종으로 포팅.
기존 TurnSubsystem/StageSubsystem에 글로벌 덱 연동을 추가하고, SpawnPawns를 DeckSubsystem 편성 기반으로 교체하였다.

---

## 주요 구현 상세

### 1. UPlayerDataSubsystem
- Gold (초기 1000), DifficultyLevel (초기 1)
- AddGold, SpendGold(bool), HasEnoughGold, IncrementDifficulty, ResetAll

### 2. UPawnSubsystem
- TArray<UDPawn*> PawnList
- AddPawn, RemovePawn, GetPawns

### 3. UItemSubsystem
- TArray<UDEquipment*> EquipList
- AddEquip, RemoveEquip, GetEquips

### 4. UDeckSubsystem

**A. Pawn 편성 슬롯:**
- PawnSlots[5] (초기 2개 오픈)
- ExpandSlot, AssignPawnToDeck, UnassignPawnFromDeck, GetActivePawns

**B. 글로벌 카드 덱:**
- DeckCardList → CardPool → HandCardList / DiscardPool
- 상수: MaxHandSize=10, InitialHandSize=5, DrawPerTurn=1
- InitStage (셔플+5장), StartTurnDraw (1장/턴), DrawCards, UseCard
- Supply/Consume 카드는 사용 시 폐기, Stable은 DiscardPool로

### 5. TurnSubsystem 연동
- StartPawnTurn의 bDrawCards 블록에서 글로벌 덱 드로우 1회 추가
- 기존 Pawn별 개인 DrawCards는 유지

### 6. StageSubsystem 연동
- UseCardSelf/TryUseCard: CardOwner가 null이면 DeckSubsystem::UseCard 호출
- LoadStage: SpawnPawns 후 DeckSubsystem::InitStage 호출
- SpawnPawns: DeckSubsystem::GetActivePawns 기반으로 교체 (ResetStats+BuildCardPool+InitHand)
- ProcessNextEffect DrawCard: 폰 개인풀 + 글로벌 덱 양쪽 드로우

## 아키텍처 매핑

```
Unity MonoSingleton          →  UE5 UGameInstanceSubsystem
──────────────────────────────────────────────────
PlayerManager                →  UPlayerDataSubsystem
PawnManager                  →  UPawnSubsystem
ItemManager                  →  UItemSubsystem
DeckManager                  →  UDeckSubsystem
```

### 카드 덱 흐름 (글로벌)
```
DeckCardList ─→ [InitStage: 셔플] ─→ CardPool ─→ [DrawCards] ─→ HandCardList
                                          ↑                          │
                                   RefillFromDiscard            [UseCard]
                                          ↑                          │
                                    DiscardPool ←── Stable 카드 ─────┘
                                                    Supply/Consume → 폐기
```

### 카드 덱 흐름 (Pawn 개인)
```
장비.CardId + GrowthCardIds ─→ [BuildCardPool: 셔플] ─→ PawnCardPool ─→ PawnHandCards
                                                              ↑              │
                                                       RefillFromDiscard  [UsePawnCard]
                                                              ↑              │
                                                        PawnDiscardPool ←────┘
```
