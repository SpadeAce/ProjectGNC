# 데이터 툴 — 누락 컬럼 기본값 처리 및 Asset 필드 보존

- **일시**: 2026-03-29
- **분류**: 기능 개선 (데이터 파이프라인)
- **범위**: Tools/validator.py, Tools/json_exporter.py, Tools/main.py

---

## 배경

Excel → Protobuf → JSON → UE5 DataTable 파이프라인에서 두 가지 문제가 있었다.

1. **스키마에 정의된 컬럼이 데이터 엑셀에 없으면 ValidationError 발생** — 새 필드를 스키마에 추가했지만 아직 데이터를 채우지 않은 경우 파이프라인이 중단됨
2. **`asset:X` 타입 필드가 엑셀에 없을 때 기존 값 소실** — UE5 에디터에서 수동 설정한 에셋 경로(예: TileMapPreset)가 파이프라인 재실행 시 빈 값으로 덮어씌워짐. `fill_data_table_from_json_string()`이 전체 교체 방식이기 때문.

## 변경 내용

### 1. validator.py — 누락 컬럼 에러 → 경고 (302~306행)

| 변경 전 | 변경 후 |
|---------|---------|
| `raise ValidationError("스키마에 있지만 시트에 없는 컬럼: ...")` | `print("경고: ... → 기본값(proto3 zero value)으로 채워집니다.")` |

누락된 컬럼은 proto3 기본값으로 처리된다: int→0, string→"", bool→false, enum→0번 값, repeated→빈 배열.

### 2. json_exporter.py — Asset 필드 보존 로직 추가 (101~139행)

새 함수 2개 추가:
- `_load_existing_json(json_path)`: 기존 JSON 파일을 `{row_name: row_dict}` 맵으로 로드
- `_preserve_asset_fields(rows, existing_map, asset_field_names, table_name)`: 새 데이터에서 asset 필드가 비어있으면(`""`, `[]`, `None`) 기존 JSON의 동일 행(Name 키 매칭)에서 값을 복원

`export_bytes_to_json()` 시그니처에 `asset_fields: dict[str, list[str]] | None` 파라미터 추가. JSON 파일 저장 직전에 보존 로직을 실행한다.

### 3. main.py — Asset 필드 정보 전달 (188~195행)

Step 8에서 `excel_data.schema`를 순회하여 `asset:X` 타입인 필드명을 테이블별로 추출하고, `export_bytes_to_json()`에 `asset_fields` 인자로 전달한다.

## 동작 흐름

```
엑셀(asset 컬럼 없음)
  → validator: 경고 출력, 통과
  → serializer: proto3 기본값(빈 string)
  → json_exporter:
      1. 기존 JSON 로드 → {row_name: row} 맵
      2. 새 행의 asset 필드가 비어있으면 기존 값으로 교체
      3. 콘솔에 [PRESERVE] 로그 출력
  → ImportDataTables: fill → 에셋 경로 유지됨
```

## 영향 범위

- 현재 `asset:X` 타입 필드: `StagePresetData.presetPath` (`asset:TileMapPreset`)
- 향후 asset 타입 필드가 추가되면 자동으로 보존 대상에 포함됨 (스키마 기반 자동 감지)
- 엑셀에 asset 컬럼이 **존재하는** 경우에는 새 값이 정상 반영됨 (기존 동작 유지)
