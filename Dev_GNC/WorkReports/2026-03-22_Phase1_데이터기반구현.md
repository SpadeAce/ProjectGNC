# 작업 보고서: Phase 1 — 데이터 기반 구현

**날짜**: 2026-03-22
**작업자**: Claude

## 요청 내용

포팅 가이드(Phase 1: 데이터 기반)에 따라, CardGame의 전체 Enum, 데이터 테이블 구조체 12종, 데이터 서브시스템 2종, 데이터 오브젝트 계층(DObject→DPawn/DMonster/DCard/DEquipment), Protobuf→JSON 변환 스크립트를 구현하였다.

## 수행한 작업

- Unity EnumContainer.cs + GameData.proto의 모든 enum을 UENUM(BlueprintType)으로 변환 (15종)
- GameData.proto의 12개 message를 FTableRowBase 구조체로 1:1 매핑
- FBuffEntry, FPawnStats 보조 USTRUCT 정의
- UDataSubsystem (UGameInstanceSubsystem): 11개 DataTable 로드, ID/Level 기반 조회, 2차 인덱스
- UTextSubsystem (UGameInstanceSubsystem): 로컬라이제이션 텍스트 조회, 언어 설정 저장
- DObject 계층 5개 UCLASS 구현 (UDObject, UDItem, UDCard, UDEquipment, UDPawn, UDMonster)
- UDPawn: 전투(데미지/회복/보호막), 버프, 레벨링, 장비, 카드핸드(드로우/사용/셔플) 전체 로직
- UDMonster: 전투, 버프 로직 (DPawn과 동일 패턴)
- ProtoToJSON.py: Unity .bytes → UE5 DataTable JSON 변환 스크립트
- UE5 빌드 성공 확인

## 주요 결정 사항

- **CSV 대신 JSON 임포트**: proto의 repeated 필드(TArray)가 있는 테이블이 6개 → UE5 CSV 임포트는 TArray 미지원이므로 JSON 포맷 채택
- **델리게이트 공유**: FOnStatsChanged, FOnFloatingText를 DPawn.h에서 선언하고 DMonster.h에서 include하여 재사용 (순환 의존 없음)
- **UObject 초기화 패턴**: NewObject<T>(Outer) → InitFromId(Id, DataSub) 2단계 (UObject는 파라미터 생성자 불가)
- **PawnGrowth 복합키**: ClassType * 1000 + Level을 TMap 키로 사용하여 O(1) 조회
- **포팅 가이드 네이밍 준수**: DataSubsystem/TextSubsystem (CardGame prefix 생략), CardGameRowTypes 등 기존 가이드 컨벤션 유지
