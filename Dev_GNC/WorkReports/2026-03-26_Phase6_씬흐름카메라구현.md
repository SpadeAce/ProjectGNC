# Phase 6: 씬 흐름 + 카메라 구현

> **날짜**: 2026-03-26
> **작업 유형**: 기능 구현 (Unity→UE5 포팅)
> **Phase**: Phase 6 — 씬 흐름 + 에셋 (C++ 코드 범위)

---

## 작업 요약

Unity의 SceneController/SceneBase/CameraController를 UE5 C++로 포팅했다.
공통 GameMode(ACardGameMode)와 전투 전용 GameMode(ACardStageGameMode), PlayerController, 탑다운 카메라 폰을 신규 생성하고, Phase 5에서 스텁 처리했던 씬 전환 핸들러 3곳(PausePopup/BattleResultPopup/LobbyPage)을 실제 `OpenLevel` 호출로 교체하여 Title→Lobby→Stage→(결과)→Lobby의 전체 게임 루프를 완성했다.

## 신규 파일 (8개)

### 1. ACardGameMode (`CardGameMode.h/cpp`)
- AGameModeBase 상속, 공통 베이스
- 생성자에서 HUDClass=ACardGameHUD, PlayerControllerClass=ACardGamePlayerController, DefaultPawnClass=ACardGameCameraPawn 설정
- Title/Lobby 맵에서 BP 파생 없이 직접 사용 가능
- Unity SceneBase 대응

### 2. ACardStageGameMode (`CardStageGameMode.h/cpp`)
- ACardGameMode 상속, 전투 스테이지 전용
- BeginPlay에서:
  1. PlayerDataSubsystem→DifficultyLevel 조회
  2. DataSubsystem→GetStagePresetByLevel()로 프리셋 Row 검색
  3. PresetPath 배열에서 랜덤 선택 → UTileMapPreset DataAsset 로드
  4. StageSubsystem→LoadStage(Preset) 호출 (내부에서 InitStage/StartGame 체인 완료)
  5. GridSubsystem→GetMapCenter()로 카메라 피벗 설정
- Unity StageScript.OnEnterScene() 대응

### 3. ACardGamePlayerController (`CardGamePlayerController.h/cpp`)
- APlayerController 상속
- 생성자: bShowMouseCursor = true
- BeginPlay: FInputModeGameAndUI 설정 (마우스 커서 표시 + UI 클릭 + 월드 입력)
- Unity에서는 별도 PC가 없었으나 UE5에서는 커서/입력모드 관리에 필수

### 4. ACardGameCameraPawn (`CardGameCameraPawn.h/cpp`)
- APawn 상속, 탑다운 전략 카메라
- **컴포넌트**: USceneComponent(Pivot) → USpringArmComponent → UCameraComponent
- **입력** (Enhanced Input, BP에서 IA/IMC 연결):
  - Pan (우클릭 드래그): Yaw 기준 Forward/Right 방향으로 Pivot 이동
  - Rotate (Q/E): RotationStep(90°) 단위 Yaw 회전, 보간
  - Zoom (마우스 스크롤): TargetArmLength 조절 (300~3000)
- **설정값**: PanSpeed, RotationInterpSpeed, RotationStep, MinZoom/MaxZoom/ZoomSpeed, CameraElevation(45°)
- **메서드**: SetPivot(FVector) — StageGameMode에서 맵 중앙으로 설정
- Unity CameraController.cs 대응

## 설계 결정

### GameInstance 커스텀 불필요
- 포팅 가이드에서 `UGameInstance 레벨 전환`이 명시되어 있으나, 실제로는 기본 UGameInstance에 이미 6개 GameInstanceSubsystem이 등록되어 동작
- 레벨 전환은 `UGameplayStatics::OpenLevel()`로 충분 (Unity SceneController.ChangeScene 대응)
- 서브시스템 리셋은 각 호출부에서 직접 수행 (5줄 중복이나 별도 유틸리티 클래스까지는 불필요)

### 씬 전환 시 UI 정리 불필요
- OpenLevel 호출 시 현재 World가 파괴되므로, HUD·위젯·팝업이 자동 정리됨
- 기존 CloseAllPopups/CloseAllPages 호출 제거

### CameraPawn vs PlayerController 카메라
- PlayerController에 카메라 로직을 넣을 수도 있으나, UE5 패턴상 Pawn에 카메라 컴포넌트를 두는 것이 표준
- SpringArm + Camera 조합으로 충돌 테스트 없는 자유 카메라 구현
- Enhanced Input 사용 (프로젝트 기존 패턴과 동일)

### StageGameMode의 역할
- Unity StageScript.OnEnterScene()의 역할을 GameMode.BeginPlay()가 대체
- StageSubsystem::LoadStage()가 이미 내부에서 GridSub→GenerateTileMap, TurnSub→StartGame, DeckSub→InitStage, Pawn→BuildCardPool/InitHand 체인을 완료하므로, GameMode는 프리셋 해석과 LoadStage 호출만 담당

## 게임 루프 흐름

```
[Title 맵]
  ACardGameMode → ACardGameHUD → UTitlePageWidget
  ├─ NewGame → ResetAllSubsystems() → OpenLevel("Lobby")
  ├─ LoadGame → OpenLevel("Lobby")
  ├─ Option → (미구현)
  └─ Exit → QuitGame

[Lobby 맵]
  ACardGameMode → ACardGameHUD → ULobbyPageWidget
  ├─ StartBattle → OpenLevel("Stage")
  ├─ DeckSetting/PawnManage/Shop/Recruit → 페이지 전환
  └─ Exit → ResetAll → OpenLevel("Title")

[Stage 맵]
  ACardStageGameMode.BeginPlay()
    → DataSub→GetStagePresetByLevel(Difficulty)
    → StageSub→LoadStage(Preset) [InitStage+StartGame 내부 체인]
    → CameraPawn→SetPivot(MapCenter)
  ACardGameHUD → UStagePageWidget
  ├─ Pause → UPausePopupWidget
  │    └─ Exit → ResetAll → OpenLevel("Title")
  └─ OnBattleEnd → UBattleResultPopupWidget
       ├─ Continue (승리) → 보상 처리 → OpenLevel("Lobby")
       └─ Exit (패배) → ResetAll → OpenLevel("Title")
```
