# C++ Row Struct 자동 생성 및 ImportDataTables classref 보존

- **일시**: 2026-03-29
- **분류**: 기능 추가 (데이터 파이프라인 + 코드 생성 + 임포트 보존)
- **범위**: Tools/ue_codegen.py, Tools/main.py, Tools/ImportDataTables.py, Tools/config.yaml

---

## 배경

두 가지 문제가 있었다.

1. **Proto 스키마에서 classref로 변경해도 C++ 구조체에 반영 안 됨** — `CardGameRowTypes.h`가 수동 관리여서, Excel 스키마의 `classref:` 타입 변경이 자동으로 C++ `TSoftClassPtr`로 변환되지 않았음

2. **ImportDataTables.py 실행 시 에디터 classref 값 초기화** — `fill_data_table_from_json_string()`이 전체 교체 방식이고, JSON에는 Unity 경로 문자열만 있어서 에디터에서 수동 설정한 BP 클래스 레퍼런스가 매번 소실됨

---

## 변경 내용

### 1. Tools/ue_codegen.py — C++ Row Struct 헤더 자동 생성 (신규)

Excel 스키마의 FieldType을 읽어 `CardGameRowTypes.h`를 자동 생성한다.

| Excel FieldType | 생성되는 C++ 타입 |
|---|---|
| `int32` | `int32` |
| `string` | `FString` |
| `bool` | `bool` (bPrefix 자동 추가) |
| `enum:X` | `EX` |
| `classref:AX` | `TSoftClassPtr<AX>` |
| `asset:UX` | `TSoftObjectPtr<UX>` |
| repeated 필드 | `TArray<위 타입>` |

추가 처리:
- `{X}IdAlias` 헬퍼 필드는 C++ 구조체에서 자동 제외 (alias 해석용)
- classref/asset 참조 클래스의 전방선언 자동 생성
- camelCase → PascalCase 변환 (특수 약어 `hp` → `HP` 지원)
- bool 필드 `b` 접두어 자동 추가 (UE5 컨벤션)
- 독립 실행 가능: `python ue_codegen.py`

### 2. Tools/main.py — 파이프라인 확장 (Step 9, 10)

| Step | 설명 |
|---|---|
| Step 9 | `ue_codegen.py` 호출 → `CardGameRowTypes.h` 자동 생성 |
| Step 10 | `_table_meta.json` 생성 → classref/asset 필드 목록 (ImportDataTables.py용) |

`_table_meta.json` 예시:
```json
{
  "MonsterData": ["prefabPath"],
  "PawnData": ["prefabPath"],
  "StagePresetData": ["presetPath"]
}
```

### 3. Tools/ImportDataTables.py — classref/asset 필드 보존

import 전 기존 DataTable에서 classref/asset 필드 값을 읽어 보존하는 흐름:

```
1. _table_meta.json 로드 → classref/asset 필드 목록 확인
2. 기존 DataTable에서 해당 필드 값 읽기 (UE5 Python API)
3. 새 JSON의 해당 필드가 비어있거나 유효한 UE5 경로가 아닌 경우 기존 값으로 병합
4. 병합된 JSON을 파일에도 저장 (다음 파이프라인의 json_exporter가 보존)
5. 병합된 JSON으로 DataTable 덮어쓰기
```

보존 조건:
- 기존 값이 `/`로 시작하는 유효한 UE5 에셋 경로인 경우에만 보존
- 새 JSON 값이 비어있거나 UE5 경로가 아닌 경우(Unity 경로 등)에만 교체

### 4. Tools/config.yaml

```yaml
ue_header_path: "../Source/Dev_GNC/CardGame/Data/CardGameRowTypes.h"
```

## 보존 체인 전체 흐름

```
[Excel 데이터] → main.py → json_exporter (이전 JSON에서 보존)
                                ↓
                          DT_*.json (classref 값 포함)
                                ↓
                    ImportDataTables.py
                    ├─ 기존 DataTable 읽기 (에디터 설정값)
                    ├─ JSON과 병합 (에디터 값 우선)
                    ├─ 병합 JSON 파일 저장 (다음 파이프라인용)
                    └─ DataTable 임포트
```

## 사용자 조치 사항

1. Excel 스키마에서 classref/asset 클래스명에 **UE5 접두어 포함** 필요:
   - `classref:CardGameActor` → `classref:ACardGameActor`
   - `asset:TileMapPreset` → `asset:UTileMapPreset`
2. `projectilePath`도 classref로 전환하려면 해당 FieldType을 `classref:ACardGameActor`로 변경
3. 수정 후 `python main.py` 실행 → C++ 헤더 자동 갱신 + 메타데이터 생성
4. UE5 에디터 재컴파일 후 `ImportDataTables.py` 실행
