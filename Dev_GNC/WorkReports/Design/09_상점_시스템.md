## 데이터 구조

### FShopDataRow (DataTable Row Struct)
```
Id       int32                — 행 고유 ID
Level    int32                — 상점 레벨
CardIds  TArray<int32>        — 진열할 카드 ID 목록
ItemIds  TArray<int32>        — (미사용, 향후 아이템 확장용)
```

### UDataSubsystem 상점 조회 방식
```cpp
FShopDataRow* GetShopByLevel(int32 Level);   // 레벨로 조회
int32 GetShopMaxLevel();                      // 최고 레벨 반환
```

## UShopPageWidget 동작 흐름

```
OnOpened()
  └─ ULobbySubsystem::ShopCards == nullptr → GenerateShopList()
                                              + RefreshItems()
  └─ RefreshGoods()   (골드 표시만 갱신)

ULobbySubsystem::GenerateShopList()
  └─ UDataSubsystem::GetShopByLevel(ShopLevel) → FShopDataRow
  └─ CardIds → FCardDataRow 수집 (최대 10개)

RefreshItems()
  └─ 기존 ShopCardItemWidget RemoveFromParent
  └─ ShopCards 기반 5×2 그리드 배치
  └─ Item->SetData(CardData), OnBuyItem Delegate 등록
```

### 카드 그리드 배치 (5열 × 2행)
```cpp
int32 Row = i / 5;
int32 Col = i % 5;
// 슬롯 위치 계산
```

## UShopCardItemWidget 동작

```cpp
SetData(FCardDataRow* Data)
  → CardData 저장
  → TextPrice = Data->Price > 0 ? FString::FromInt(Data->Price) : "FREE"

HandleBuyClicked()
  → NewObject<UDCard>() 생성
  → UDeckSubsystem::AddCard(Card)
  → SetVisibility(Collapsed)   // 숨김 처리
  → OnBuyItem Delegate 발생
```

## 상점 레벨 시스템

- 최초 레벨: `ULobbySubsystem::ShopLevel = 1`
- 최고 레벨: `UDataSubsystem::GetShopMaxLevel()` (데이터 기반 동적 산출)
- LevelUp 버튼 → `ULobbySubsystem::LevelUpShop()` → `GenerateShopList()` → `RefreshItems()`
- 최고 레벨 도달 시 LevelUp 버튼 동작 없음

---

## 진열 유지 정책

- `ULobbySubsystem::ShopCards`가 비어있지 않으면 목록 재생성 건너뜀
- GameInstanceSubsystem이므로 레벨 전환과 무관하게 유지
- `UShopPageWidget::Refresh()` 공개 메서드: Stage 종료 후 외부에서 호출 → `ULobbySubsystem::GenerateShopList()` 호출
