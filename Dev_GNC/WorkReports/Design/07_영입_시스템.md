# 영입 시스템 (Recruit)

## 구현 상태: 완성 (재화 차감, ULobbySubsystem 데이터 분리 완료)

---

## 핵심 파일

| 파일 | 역할 |
|------|------|
| `UI/RecruitPageWidget.h/cpp` | 영입 페이지 — UI 렌더링, 이벤트 처리 |
| `Subsystem/LobbySubsystem.h/cpp` | 영입 후보 목록 보관 (페이지 독립) |
| `Subsystem/DeckSubsystem.h/cpp` | 영입된 Pawn을 덱에 추가 (`AddPawn`), 골드 관리 |
| `Subsystem/DataSubsystem.h/cpp` | 전체 FPawnDataRow 목록 조회 |

---

## 영입 흐름

```
URecruitPageWidget::OnOpened()
    └── ULobbySubsystem::RecruitPawns가 비어있으면 GenerateRecruitList()
            ├── UDataSubsystem → 전체 FPawnDataRow 목록
            └── 랜덤 3개 선택 → NewObject<UDPawn>() (CodeName 자동 할당)
    └── RefreshItems()
            ├── 기존 아이템 위젯 전부 RemoveFromParent
            └── RecruitPawns 각각 → 아이템 위젯 생성 → SetData(Pawn)

아이템 영입 버튼 클릭
    ├── UDeckSubsystem::AddPawn(Pawn)
    ├── UPawnSubsystem::AddPawn(Pawn)
    └── OnRecruited Delegate 발생 → RecruitPageWidget::OnItemRecruited()
            ├── RecruitPawns에서 제거
            └── 아이템 위젯 RemoveFromParent
```

---

## URecruitPageWidget 주요 멤버

| 멤버 | 설명 |
|------|------|
| `RefreshItems()` | 기존 아이템 제거 후 `ULobbySubsystem::RecruitPawns` 기반으로 재생성 |
| `RefreshGoods()` | 보유 골드 텍스트 갱신 |
| `Refresh()` | 외부에서 목록 강제 갱신 → `ULobbySubsystem::GenerateRecruitList()` 호출 |

> 목록 데이터(`RecruitPawns`, `RecruitSlotCount`)는 `ULobbySubsystem`이 보관.

## ULobbySubsystem 영입 관련 API

| 메서드/프로퍼티 | 설명 |
|----------------|------|
| `GetRecruitPawns()` | 현재 영입 후보 목록 |
| `GenerateRecruitList()` | 전체 PawnData에서 랜덤 3개 뽑아 UDPawn 생성 |
| `RemoveRecruitPawn(Pawn)` | 영입 완료 후 후보 목록에서 제거 |

---

## 후보 목록 갱신 시점

| 시점 | 갱신 여부 |
|------|----------|
| 페이지 최초 오픈 | O (목록이 없을 때만) |
| 페이지 닫고 재오픈 | X (기존 목록 유지) |
| Reroll 버튼 클릭 | O |
| Stage 복귀 | O (`Refresh()` 외부 호출 필요 — 미연결) |

void AddPawn(UDPawn* Pawn);              // 영입된 Pawn을 덱에 추가
