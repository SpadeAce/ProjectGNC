# 전투 시스템

## 구현 상태: 기본 완성 + 명중율 시스템 (AI 고도화 미구현)

---

## 핵심 파일

| 파일 | 역할 |
|------|------|
| `Subsystem/TurnSubsystem.h/cpp` | 턴 진행, 몬스터 AI, 공격 처리 |
| `Subsystem/StageSubsystem.h/cpp` | 액터 관리, 타일 선택, 승패 판정 |
| `CardStageGameMode.h/cpp` | 스테이지 진입/종료 |

---

## 턴 구조

```
StartGame()
    └──▶ [Pawn 턴]
              플레이어 행동 (이동, 카드 사용)
              EndPawnTurn() 버튼 클릭
              └──▶ [Monster 턴] (자동)
                        Alert 판정
                        각 몬스터 행동 순서대로 실행
                        모든 몬스터 완료
                        └──▶ [Pawn 턴] (카드 1장 드로우)
```

---

## Pawn 턴

### 이동
- 타일 클릭 → `UStageSubsystem::SelectTile()`
- 선택된 Pawn → 이동 범위 하이라이트 (BFS)
- 빈 타일 클릭 → A* 경로탐색 → `ACardGameActor::MoveTo()`
- 이동 후 잔여 이동력 기록 (`UTurnSubsystem::RemainingMovement`)
- 이동력 0이 되면 이동 불가

### 카드 사용
- UStagePageWidget에서 카드 드래그&드롭으로 사용
- 자세한 내용은 `03_카드_시스템.md` 참고

### 턴 종료
- EndTurn 버튼 → `UTurnSubsystem::EndPawnTurn()`
- `UStageSubsystem::Deselect()` 호출 → 선택 해제
- Monster 턴으로 전환

## Monster 턴

### Alert 상태 판정
- 어느 한 몬스터의 Sight 범위 내에 Pawn이 1명이라도 있으면 **전체 Alert**
- Alert 상태: 모든 몬스터가 Pawn을 추적
- 비Alert 상태: 모든 몬스터가 배회(Wander)

### 몬스터 행동 순서 (Alert 상태)
```
1. 가장 가까운 Pawn 탐색
2. 이동 전 공격 시도 (사정거리 내이면 공격 후 행동 종료)
3. 아니면 Pawn 방향으로 이동
4. 이동 후 공격 시도 (사정거리 내이면 공격)
```

### 몬스터 행동 순서 (비Alert 상태 - 배회)
```
이동력 최대 2칸 제한으로 랜덤 이동
```

### 공격 처리 (`ExecuteAttack`)
```
1. 공격 모션 (Attack01 AnimMontage)
2. 피해 적용: target.TakeDamage(attackPower)
   - 실피해 = attackPower - target.Armor
   - Shield 우선 감소 → HP 감소
3. GetHit 모션
4. 사망 체크 → Die() 애니메이션
5. UStageSubsystem::CheckBattleResult()
```

## 명중율 시스템

### 개요
- FPawnDataRow의 `Accuracy` 컬럼으로 명중 확률 결정 (만분율: 10000 = 100%)
- 장비/성장 보너스로 합연산 (예: 기본 50% + 장비 5% = 55%)
- **Enemy 타겟에만 적용** — Ground, Ally, Self 타겟은 무조건 명중

### 핵심 파일

| 파일 | 역할 |
|------|------|
| `Data/FPawnStats.h` | Accuracy 스탯 합산 (PawnData + 장비 + 성장) |
| `Data/DPawn.h/cpp` | Accuracy 런타임 프로퍼티 |
| `Subsystem/StageSubsystem.h/cpp` | 명중 판정 (ApplyCardEffects) |
| `Grid/CardGameActor.h/cpp` | Evade 애니메이션 재생 |
| `UI/StagePageWidget.h/cpp` | 드래그 중 명중율 % 표시 |

### 명중 판정 흐름
```
ApplyCardEffects 진입
  → 조건: card.Target == Enemy && caster == UDPawn && target == UDMonster
  → FMath::RandRange(0, 9999) < pawn.Accuracy → 명중
  → 카드 당 1회 판정, 모든 효과에 적용

명중 시:
  → 기존 로직 (TakeDamage + ReceiveHit + 사망 처리)

빗나감 시:
  → Damage: 피해 미적용 + Evade 애니메이션 + "MISS" 플로팅 텍스트
  → Debuff: 미적용 (스킵)
  → Heal/Shield/Buff 등: 정상 적용 (명중율 무관)
```

### UI 표시
- 카드 드래그 중 Enemy 위에 마우스 호버 시 명중율 표시 (0~100%)
- Ground 타겟이거나 대상이 없으면 숨김

## 승패 판정

- **위치**: `UStageSubsystem::CheckBattleResult()`
- **Delegate**: `UStageSubsystem::OnBattleEnd` (UStagePageWidget이 구독)
- **승리 조건**: EnemyActor TMap 내 모든 ACardGameActor가 사망 상태
- **패배 조건**: UserActor TMap 내 모든 ACardGameActor가 사망 상태
- **호출 시점**:
  1. `UTurnSubsystem::ExecuteAttack()` — 사망 애니메이션 완료 후
  2. `UStageSubsystem::ApplyCardEffects()` — 카드 데미지로 몬스터 사망 후

### 승리 시 보상 흐름

```
몬스터 처치 (ApplyCardEffects / UTurnSubsystem::ExecuteAttack)
    → StageGoldReward += monster.Data.Gold
    → StageExpReward  += monster.Data.Exp

CheckBattleResult() — 승리 판정
    → StageGoldReward += 100  (클리어 보너스)
    → OnBattleEnd.Broadcast(true)
        → UStagePageWidget::OnBattleEnd()
            → BattleResultParam { bIsWin=true, GoldReward, ExpReward }
                → UBattleResultPopupWidget::PreOpen()
                    → 골드/경험치 텍스트 표시
                    → 카드 보상 3장 세팅 (가중치 랜덤)
                → HandleContinueClicked()
                    → UPlayerDataSubsystem::AddGold(GoldReward)
                    → UPlayerDataSubsystem::IncrementLevel()
                    → DeckPawns 전원 AddExp(ExpReward)
                    → 선택된 카드 → UDeckSubsystem::AddCard()
```

- 패배 시 `GoldReward = 0`, `ExpReward = 0` 전달 → 보상 미지급

### 카드 보상 선택 (UBattleResultPopupWidget)

```
승리 시:
    1. UPlayerDataSubsystem::DifficultyLevel로 StageRewardData 조회
    2. CardId / CardProb (1:1 매칭) 에서 가중치 랜덤 3장 선출 (중복 불가)
    3. BattleResultItem × 3 에 카드 정보 세팅
    4. 플레이어가 1장 선택 (토글 — 재클릭 시 해제)
    5. Continue 클릭 → 선택 카드 UDeckSubsystem::AddCard()에 추가
       (미선택 시 카드 보상 없이 진행)
```

## 액터 관리 (UStageSubsystem)

| TMap | 내용 |
|------|------|
| `UserActorMap` | 플레이어 Pawn (int32 ID → ACardGameActor*) |
| `EnemyActorMap` | 몬스터 (int32 ID → ACardGameActor*) |

- Actor 사망: `ACardGameActor::Die()` → bIsDead 플래그 + `Slot->ClearEntity()` + 숨김 처리
- 생존 판단: `bIsDead` 플래그 기준
