## 카드 데이터 구조

### FCardDataRow (DataTable Row Struct)
| 필드 | 타입 | 설명 |
|------|------|------|
| Id | int32 | 카드 고유 ID |
| Type | ECardType | 카드 유형 |
| Target | ETargetType | 사용 대상 |
| Name | FString | 카드 이름 |
| Desc | FString | 카드 설명 |
| EnergyCost | int32 | 공용 행동력(SharedActingPower) 소모 비용 |
| AmmoCost | int32 | Pawn 개별 탄약(Ammo) 소모 비용 |
| Price | int32 | 판매 가격 |
| Range | int32 | 사용 가능 사거리 (카드 Range + Pawn Range 합산) |
| Radius | int32 | 효과 적용 반경 (0=단일 대상, 1=1칸 주변까지) |
| EffectIds | TArray\<int32\> | 적용할 효과 ID 목록 |
| EffectValues | TArray\<int32\> | 효과값 목록 (EffectIds와 1:1 대응) |
| IconPath | FString | 아이콘 리소스 경로 |

### ECardType
| 값 | 이름 | 설명 | 사용 후 처리 |
|----|------|------|------------|
| 1 | Stable | 지속 카드 | 소모풀로 반환 |
| 2 | Supply | 보급품 | 소멸 |
| 3 | Consume | 소모품 | 소멸 |

### ETargetType
| 값 | 이름 | 설명 |
|----|------|------|
| 0 | None | 대상 없음 |
| 1 | Self | 시전자 자신 |
| 2 | Enemy | 적 Actor (UDMonster) |
| 3 | Ground | 특정 타일 위치 (walkable 타일) |
| 4 | Ally | 아군 Pawn (UDPawn) |

## 버프/디버프 시스템

카드 효과 BuffAttack~DebuffMovement는 `FBuffEntry` 구조체로 관리된다.

### FBuffEntry (`Data/BuffEntry.h`)
```cpp
USTRUCT(BlueprintType)
struct FBuffEntry
{
    GENERATED_BODY()
    ECardEffectType Type;
    int32 Value;           // 적용된 델타 (+버프, -디버프)
    int32 RemainingTurns;
};
```

### 지속 시간 규칙
- **사용한 턴도 지속시간에 포함** — Duration=2, 1턴 획득 → 1,2턴 유지, 3턴 시작 시 제거
- `TickBuffs()`: 먼저 RemainingTurns 감소 → 0 이하이면 스탯 원복 후 제거

### 적용/만료 흐름
```
카드 사용 → UStageSubsystem::ApplyCardEffects()
    → UDPawn/UDMonster::ApplyBuff(Type, Value, Duration)
        → Buffs 배열에 추가
        → Attack/Armor/Movement 직접 += Value

Pawn 턴 시작 → UTurnSubsystem::StartPawnTurn()
    → foreach UserActors: pawn->TickBuffs()
    → foreach EnemyActors: monster->TickBuffs()
        → RemainingTurns 감소 → 0 이하이면 스탯 -= Value, 배열에서 제거
```

### 관련 메서드 (UDPawn / UDMonster 공통)
| 메서드 | 설명 |
|--------|------|
| `ApplyBuff(Type, Value, Duration)` | 버프 적용 + 즉시 스탯 변경 |
| `TickBuffs()` | 턴 시작 시 호출 — 만료된 버프 제거 및 스탯 원복 |
| `GetBuffs()` | 현재 활성 버프 목록 |

## 자원 연동

카드 사용 시 `EnergyCost`와 `AmmoCost`를 각각의 자원에서 차감한다.
→ 자세한 내용은 `08_자원_시스템.md` 참고

```
카드 사용 흐름:
1. CanAffordCard(Card, Pawn) 검증 → SharedActingPower + Ammo 둘 다 충족해야 사용 가능
2. SpendCardCost(Card, Pawn)
     UTurnSubsystem::ConsumeSharedActingPower(EnergyCost) → OnSharedActingPowerChanged 발생
     Pawn->ConsumeAmmo(AmmoCost) → OnStatsChanged 발생
3. UDeckSubsystem::UseCard(Card) → OnHandChanged 발생 → UI 갱신
4. ApplyCardEffects(...)
```

> **순서 주의**: `SpendCardCost`를 `UseCard` 이전에 호출해야
> `SetCardList()` 실행 시점에 올바른 자원 수치로 disabled 상태가 갱신된다.

## 덱 관리 (UDeckSubsystem)

### 구조
```
덱 구성
├── DeckPawnList   : Pawn 목록
├── DeckCardList   : 덱의 카드 전체 목록
└── DeckEquipment  : 장비 (현재 미사용)

런타임 풀 (공유)
├── CardPool       : 드로우 대기 풀 (셔플된 상태)
├── DiscardPool    : 소모된 카드 대기 풀
└── HandCardList   : 글로벌 핸드 (최대 10장)
```

> Pawn 개인 핸드는 `UDPawn::PawnHandCards`에 저장. Pawn 전용 카드풀(`PawnCardPool`)을 보유하며, 장착 장비의 카드로 구성. → `05_캐릭터_시스템.md` 참고

### 상수
| 상수 | 값 | 설명 |
|------|----|------|
| InitialHandSize | 5 | 스테이지 시작 초기 글로벌 핸드 |
| MaxHandSize | 10 | 글로벌 핸드 최대 크기 |
| DrawPerTurn | 1 | 턴 시작 글로벌 드로우 수 |
| InitialPawnHandSize | 3 | Pawn 초기 핸드 크기 |
| MaxPawnHandSize | 5 | Pawn 핸드 최대 크기 |
| PawnDrawPerTurn | 1 | 턴 시작 Pawn 드로우 수 |

### 주요 메서드
| 메서드 | 설명 |
|--------|------|
| `InitStage()` | 스테이지 진입 시 풀 초기화 후 5장 드로우 |
| `StartTurnDraw()` | 턴 시작 시 1장 드로우 |
| `DrawCards(int32 Count)` | N장 드로우 (public, 카드 효과 등에서도 사용) |
| `UseCard(UDCard* Card)` | 글로벌 핸드에서 제거 후 타입에 따라 소멸/소모풀 처리 |
| `InitPawnHand(UDPawn*)` | Stage 진입 시 Pawn 초기 핸드 3장 세팅 |
| `DrawPawnCards(UDPawn*, int32)` | Pawn 핸드로 N장 드로우 |
| `UsePawnCard(UDCard*, UDPawn*)` | Pawn 핸드에서 제거 후 소멸/소모풀 처리 |

### 드로우 로직
1. `CardPool`에서 드로우
2. `CardPool` 소진 시 `DiscardPool`을 섞어서 `CardPool`로 리필
3. 둘 다 소진되면 드로우 불가
4. `HandCardList` 변경 시 `OnHandChanged` Delegate 발생

### 카드 사용 처리
- `Supply` / `Consume` 타입 → 소멸
- 그 외 → `DiscardPool`로 반환

## 카드 사용 흐름 (UStagePageWidget)

```
1. UBattleCardWidget 드래그 시작
   - 선택된 Pawn 확인
   - TargetType::None이면 드래그 취소
   - UStageSubsystem 이동 범위 숨기기
   - TargetType::Enemy/Ground/Ally면 사거리 하이라이트 표시
     (카드 Range + Pawn Range 합산, 다이아몬드 형태)

2. 드래그 중
   - 아이콘 위치 업데이트
   - Enemy/Ground/Ally 타입: 매 프레임 라인 트레이스로 호버 타일 감지
     - 사거리 내 타일 위에 있으면 → Radius 기준 반경 미리보기 표시
     - 사거리 밖이면 → 반경 미리보기 클리어

3. 드래그 종료
   - 취소 판정: 시작 위치에서 100px 이내 해제
   - 반경 미리보기 클리어
   - TargetType::Self: 위로 150px 이상 드래그
   - TargetType::Enemy: 라인 트레이스 → 적 Actor(UDMonster) 타일 판정 + 거리 검사
   - TargetType::Ground: 라인 트레이스 → walkable 타일 판정 + 거리 검사
   - TargetType::Ally: 라인 트레이스 → 아군 Pawn(UDPawn) 타일 판정 + 거리 검사

4. UseCard(Card, Caster, Target, CardOwner?)
   - SpendCardCost(Card, Pawn)
   - CardOwner != nullptr → UDeckSubsystem::UsePawnCard(Card, Pawn)
   - CardOwner == nullptr → UDeckSubsystem::UseCard(Card) (글로벌 핸드)
   - ApplyCardEffectsWithRadius(Card, Caster, CenterTile)
     - Radius=0: 중심 타일만 대상
     - Radius>0: 중심 기준 다이아몬드 범위 내 대상 전체 적용
     - Enemy: 범위 내 UDMonster만 적용
     - Ally: 범위 내 UDPawn만 적용
     - Ground: 범위 내 모든 walkable 타일 (actor null 허용)
```

### 범위 계산
```
사용 가능 범위 = Card.Range + Caster의 Pawn.Range
거리 계산: 맨해튼 거리 (|dx| + |dy|)

효과 반경 = Card.Radius
반경 형태: 다이아몬드 (맨해튼 거리 기반)
```

### 반경(Radius) 미리보기
- ASquareTile의 반경 오버레이 메시로 시각화
- `UStageSubsystem::RadiusPreviewTiles` TSet으로 추적/클리어
- `UStageSubsystem::ShowRadiusPreview(CenterTile, Card)` / `ClearRadiusPreview()`
- 드래그 종료 시 항상 클리어

## Pawn 카드 핸드

각 Pawn은 글로벌 핸드와 별개로 **개인 카드 핸드**를 보유한다.
Pawn을 선택하면 해당 Pawn의 카드가 UStagePageWidget의 PawnCardGroup에 표시되며 사용할 수 있다.

| 항목 | 글로벌 핸드 | Pawn 핸드 |
|------|-------------|-----------|
| 저장 위치 | `UDeckSubsystem::HandCardList` | `UDPawn::PawnHandCards` |
| 초기 장수 | 5장 | 3장 |
| 최대 장수 | 10장 | 5장 |
| 턴 드로우 | 1장 | 1장 |
| 표시 위치 | CardGroup | PawnCardGroup |
| 드로우 풀 | `UDeckSubsystem::CardPool` | 공유 (추후 분리 예정) |

### UDPawn 핸드 관련 멤버

| 멤버 | 설명 |
|------|------|
| `PawnHandCards` (`TArray<UDCard*>`) | 현재 핸드 카드 목록 |
| `OnPawnHandChanged` (Delegate) | 핸드 변경 시 발생 |
| `AddPawnHandCard(UDCard*)` | 카드를 핸드에 추가 + Delegate 발생 |
| `RemovePawnHandCard(UDCard*)` | 카드를 핸드에서 제거 + Delegate 발생 |

### 드로우 흐름

```
Stage 진입 (ACardStageGameMode::BeginPlay)
    ├── UDeckSubsystem::InitStage()          → 글로벌 핸드 5장 세팅
    └── foreach UserActors
            └── UDeckSubsystem::InitPawnHand(Pawn)  → Pawn 핸드 3장 세팅

턴 시작 (UTurnSubsystem::StartPawnTurn, bDrawCards=true)
    ├── UDeckSubsystem::StartTurnDraw()          → 글로벌 핸드 1장 드로우
    └── foreach UserActors
            └── UDeckSubsystem::DrawPawnCards(Pawn, 1)
```

### 카드 사용 분기

```
DragCardOwner == nullptr   → UDeckSubsystem::UseCard(Card)           [글로벌 핸드]
DragCardOwner != nullptr   → UDeckSubsystem::UsePawnCard(Card, Pawn) [Pawn 핸드]
```
