## Pawn 데이터 (UDPawn)

### 스탯 목록
| 스탯 | 설명 |
|------|------|
| HP | 현재 체력 |
| Shield | 보호막 (피해 우선 흡수) |
| Armor | 방어력 (피해 감소) |
| Movement | 이동력 |
| Attack | 공격력 |
| Range | 사거리 (카드 사정거리에 합산) |
| Sight | 시야 범위 |
| ActingPower | 행동력 (카드 사용 비용 차감, 턴 시작 시 회복) |

### 피해 계산
```
실제 피해 = AttackPower - Armor
Shield 우선 감소 → Shield 소진 후 HP 감소
```

### 스탯 합산 구조

스탯은 `FPawnStats` 구조체가 합산하여 관리한다:
```
FPawnStats::Recalculate(PawnDataRow, Equips, GrowthBonuses)
    → 기본값 = FPawnDataRow.*
    → 장비 StatusType/StatusValue 쌍을 순회하여 보너스 누산
    → 성장 보너스 StatusType/Value 쌍을 순회하여 보너스 누산
    → UDPawn::UpdateStatus()가 Stats.* 값을 런타임 필드에 복사
```

런타임 변동(피해, 버프)은 UDPawn이 직접 관리하며, `ResetStats()` 시 Stats 기저값으로 복원.

### 초기화
```cpp
UDPawn::Init(int32 PawnId)
    → FPawnDataRow 로드
    → Stats.Recalculate(Data, Equips, GrowthBonuses)  // 장비 없으면 기본값만
    → UpdateStatus()                                    // 런타임 필드에 복사
```

### Delegate
- `OnStatsChanged` — HP/Shield 변경 시 발생 (HUD 업데이트용)

## ACardGameActor (인스턴스)

### 상속 구조
```
AActor
    └── AGridEntity         (Data, Slot 보유)
            └── ACardGameActor   (애니메이션, 이동)
```

### 초기화
```cpp
Actor->Init(UDPawn* Pawn)     // Pawn으로 초기화
Actor->Init(UDMonster* Mon)   // Monster로 초기화
```
- UHUDActorWidget 생성 및 연결
- `Data->OnStatsChanged` 구독

### 이동 (MoveTo)
```
속도: MoveSpeed = 3f (상수)
경로: TArray<ASquareTile*> (SmoothPath 적용된 웨이포인트)
처리:
    1. 현재 Slot->ClearEntity()
    2. 각 웨이포인트 순서대로 선형 이동
    3. 도착 Slot->SetEntity(this)
    4. OnComplete 콜백 호출
```

### 애니메이션 상태
| 상태 이름 | 설명 |
|---------|------|
| Attack01 | 공격 AnimMontage |
| GetHit | 피격 AnimMontage |
| Evade | 회피 AnimMontage |
| Die | 사망 AnimMontage → 숨김 처리 |

### 사망 처리
```
Die(OnComplete)
    → "Die" AnimMontage 재생
    → 애니메이션 완료 대기
    → Slot->ClearEntity()
    → bIsDead = true, SetActorHiddenInGame(true)
    → OnComplete 호출
```

## 장비 시스템 (UDEquipment)

```cpp
UCLASS()
class UDEquipment : public UDObject
{
    int32 EquipmentId;
    FEquipmentDataRow* Data;  // 이름, 슬롯타입, 아이콘 경로 등
};
```

`UDPawn`에 장비 목록이 있으며 장착/해제 메서드 구현 완료:
```cpp
FPawnStats Stats;
TArray<UDEquipment*> Equips;

void Equip(UDEquipment* Equipment)     // 추가 후 RecalculateStats() 호출
void Unequip(UDEquipment* Equipment)   // 제거 후 RecalculateStats() 호출
```

장착/해제 시 `FPawnStats::Recalculate()` → `UpdateStatus()` → `OnStatsChanged` 순으로 자동 갱신.

`UItemSubsystem`이 보유 장비 목록을 관리:
```cpp
TArray<UDEquipment*> EquipList;
void AddEquip(UDEquipment* Equip);
```

### 장착 상태 추적 (UDEquipment)

```cpp
int64 EquippedPawnId = 0;  // 0 = 미장착
bool IsEquipped() const { return EquippedPawnId != 0; }
void SetEquippedPawn(int64 PawnInstanceId);
void ClearEquippedPawn();
```

`UDObject::InstanceId`를 활용하여 어느 Pawn에 장착됐는지 추적.
UPawnManagePageWidget의 장비 목록은 `IsEquipped() == false`인 장비만 표시.

### 장비 슬롯 타입 (EEquipSlotType)
| 값 | 이름 | 설명 |
|----|------|------|
| 1 | WeaponMain | 주무기 |
| 2 | WeaponSub | 보조무기 |
| 3 | Armor | 방어구 |
| 4 | Helmet | 헬멧 |
| 5 | Tool | 도구 |

### 장비 스탯 (EStatusType)

`FEquipmentDataRow.StatusType` / `StatusValue`는 배열 쌍으로 정의되며, `FPawnStats`가 합산한다:

| EStatusType | 매핑 스탯 |
|------------|-----------|
| StatAtk | Attack |
| StatDef | Armor |
| StatHp | Hp (최대 HP 증가) |
| StatShield | Shield |
| StatMovement | Movement |
| StatRange | Range |
| StatCardCap | MaxHandSize 보너스 |
| StatAccuracy | (예비, 미사용) |

## Pawn 성장 시스템

### 개요
스테이지에서 몬스터를 처치하여 경험치를 획득하고, 임계값 도달 시 레벨업하는 시스템.
Gold 보상과 동일하게 스테이지 종료 시 일괄 지급한다.

### 레벨/경험치 (UDPawn)
```cpp
int32 Level = 0;  // 초기 레벨 0
int32 Exp = 0;
```

### 경험치 흐름
```
몬스터 처치 (UStageSubsystem::ApplyCardEffects)
    → StageExpReward += Monster->Data.Exp

스테이지 승리 (UStagePageWidget::OnBattleEnd)
    → BattleResultParam.ExpReward = UStageSubsystem::StageExpReward

UBattleResultPopupWidget::HandleContinueClicked
    → UDeckSubsystem::DeckPawns 순회
    → Pawn->AddExp(ExpReward)  // 참여 Pawn 전원에게 동일 경험치 지급
```

### 레벨업 처리 (UDPawn::TryLevelUp)
```
while true:
    FPawnGrowthRow = DataSubsystem->GetGrowthByClassLevel(ClassType, Level + 1)
    없거나 Exp < RequireExp → break
    Level++
    StatusType[]/StatusValue[] → GrowthBonuses에 누적
    CardId[] → GrowthCardIds에 누적
    RecalculateStats() 호출 (스탯 즉시 반영)
```

- 다중 레벨업(한 번에 여러 레벨) 지원
- `GrowthBonuses`: FPawnStats::Recalculate에 전달되어 기저 스탯에 합산
- `GrowthCardIds`: BuildCardPool 시 장비 카드 이후에 추가 (다음 스테이지부터 반영)

### FPawnGrowthRow 구조
| 필드 | 설명 |
|------|------|
| `ClassType` | 적용 대상 클래스 (Scout/Assult/Heavy/Specialist) |
| `Level` | 이 행이 부여되는 레벨 번호 |
| `RequireExp` | 레벨업에 필요한 누적 경험치 |
| `StatusType[]` | 부여할 스탯 종류 배열 |
| `StatusValue[]` | 각 스탯의 증가량 배열 |
| `CardId[]` | 영구 추가되는 카드 ID 배열 |

### Pawn 개별 카드풀 (UDPawn)

각 Pawn은 장착 장비의 `FEquipmentDataRow.CardId` 목록으로 자신만의 카드풀을 보유한다.

```cpp
static constexpr int32 InitialHandSize = 2;  // Stage 시작 시 드로우 장수
int32 GetMaxHandSize() const;  // 3 + Stats.CardCap (장비 보너스 포함)
static constexpr int32 DrawPerTurn = 1;       // 턴당 드로우 장수

void BuildCardPool();       // 장착 장비 + 성장 카드 기반 카드풀 생성 + 셔플
void InitHand();            // 초기 핸드 드로우 (InitialHandSize)
void DrawCards(int32 Count); // 드로우, 풀 고갈 시 소모풀 재충전
void UsePawnCard(UDCard* Card); // 카드 사용, Supply/Consume은 소멸, 나머지 소모풀로
```

- **카드풀 생성**: ACardStageGameMode::BeginPlay에서 배치 후 `Pawn->BuildCardPool()` + `Pawn->InitHand()` 호출
- 성장 카드(`GrowthCardIds`)는 레벨업 시 추가되어 다음 스테이지 `BuildCardPool()` 시 반영
- **턴 드로우**: `UTurnSubsystem::StartPawnTurn`에서 `Pawn->DrawCards(DrawPerTurn)` 호출
- **소모풀 재충전**: DiscardPool → CardPool 셔플 (자동)
- 장비 미장착 Pawn은 카드풀이 비어 핸드 0장

## Pawn 배치 시스템

스테이지 진입 시 `UDeckSubsystem::DeckPawns`의 Pawn을 `UTileMapPreset`의 Player 스폰 포인트에 자동 배치한다.

### UTileMapPreset 배치 관련 필드

| 필드 | 타입 | 설명 |
|------|------|------|
| `TurnLimit` | int32 | 턴 제한 (0 = 무제한) |
| `MaxPawnCount` | int32 | 배치 가능한 최대 Pawn 수 (0 = 스폰 포인트 수만큼) |
| `Tiles[i].SpawnPoint` | ESpawnPointType | Player / Enemy / None |
| `Tiles[i].SpawnId` | int32 | Enemy 전용 Monster ID (Player 스폰에서 무시) |

### 배치 수 결정 공식

```
배치 수 = min(MaxPawnCount, 스폰 포인트 수, DeckPawns.Num())
```

### 배치 흐름

```
ACardStageGameMode::BeginPlay()
    └── LoadStageFromPreset()
            1. UGridSubsystem::GetSpawnPoints(Player) → 배치 위치 목록
            2. UDeckSubsystem::DeckPawns → 배치할 Pawn 목록
            3. Count = min(PawnLimit, SpawnPoints, DeckPawns)
            4. for i in Count:
                    SpawnActor<ACardGameActor>()
                    Actor->Init(Pawn)
                    Tile->Slot->SetEntity(Actor)
                    UserActorMap.Add(i, Actor)
```

### UDeckSubsystem 관련 API

```cpp
TArray<UDPawn*> GetDeckPawns() const;  // 덱 Pawn 목록
void AddPawn(UDPawn* Pawn);             // 영입된 Pawn 추가
```
