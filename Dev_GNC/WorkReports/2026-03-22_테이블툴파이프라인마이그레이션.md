# 작업 보고서: 테이블 툴 파이프라인 마이그레이션

**날짜**: 2026-03-22
**작업자**: Claude

## 요청 내용

Unity 프로젝트(`c:\UnityProjects\CardGame\CardGameTable\Tool\`)에 있던 테이블 변환 파이프라인 전체를 UE5 프로젝트(`Dev_GNC/Tools/`)로 마이그레이션하여 Unity 프로젝트 의존성을 완전히 제거한다. Excel 데이터 파일도 UE5 프로젝트 내부로 이동하여 독립적으로 운영 가능하게 한다.

## 수행한 작업

### 1. Excel 데이터 파일 이동
- Unity `CardGameTable/Table/` → UE5 `Dev_GNC/TableData/`로 데이터 Excel 12개 복사
- Unity `CardGameTable/Proto/` → UE5 `Dev_GNC/TableData/Proto/`로 스키마 Excel 12개 + Enum.xlsx 복사
- 사용자 요청에 따라 접근성이 좋은 `Dev_GNC/TableData/` 경로에 배치

### 2. 파이프라인 Python 모듈 마이그레이션
Unity `CardGameTable/Tool/`에서 다음 모듈을 `Dev_GNC/Tools/`로 복사:
- `excel_parser.py` — Proto_*.xlsx 스키마 + *.xlsx 데이터 파싱
- `validator.py` — 스키마/데이터 검증
- `proto_generator.py` — .proto 파일 생성 + 스냅샷 비교
- `cs_generator.py` — protoc 실행
- `data_serializer.py` — .bytes 직렬화
- `templates/proto_template.j2` — Jinja2 프로토 템플릿

### 3. 통합 파이프라인 구축
- `main.py` 작성: 8단계 파이프라인 오케스트레이터
  1. config.yaml 로드 → 2. Excel 파싱 → 3. 검증 → 4. 스냅샷 비교 → 5. .proto 생성 → 6. protoc 실행 → 7. .bytes 직렬화 → 8. JSON 변환
- `json_exporter.py` 작성: Step 8 전용 모듈 (.bytes → UE5 DataTable JSON)
- `config.yaml` 작성: UE5 전용 설정 (경로를 `../TableData/` 기준으로 설정)
- `requirements.txt` 작성: openpyxl, jinja2, protobuf, pyyaml
- `RunTableTool.bat` 작성: 원클릭 실행 배치 파일 (Python/패키지 자동 확인)

### 4. protoc 바이너리 복사
- Unity `CardGameTable/Tool/protoc-34.0-win64/` → UE5 `Dev_GNC/Tools/protoc-34.0-win64/`

### 5. protobuf 7.x 호환성 수정
- `field.label` AttributeError → `MessageToDict()` API로 전환
- `including_default_value_fields` → `always_print_fields_with_no_presence`로 파라미터명 변경
- Windows cp949 콘솔에서 `✓` 문자 출력 오류 → `[OK]`로 대체

### 6. 구 파일 정리
- `Tools/ProtoToJSON.py` 삭제 (json_exporter.py + main.py로 대체)
- `Tools/RunProtoToJSON.bat` 삭제 (RunTableTool.bat으로 대체)
- `Tools/input/` 디렉터리 삭제 (파이프라인이 TableData/에서 직접 읽음)

│   ├── data_serializer.py              ← .bytes 직렬화
