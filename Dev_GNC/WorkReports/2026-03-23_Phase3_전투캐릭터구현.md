## 작업 요약

Unity C# 카드게임의 전투/턴/스테이지 시스템을 UE5 C++로 포팅.
Actor, TurnManager, StageManager를 UE5 아키텍처(AGridEntity 상속 액터 + UWorldSubsystem 2종)로 변환하였다.

## 주요 변환 사항

### 아키텍처 매핑

| Unity | UE5 | 비고 |
|-------|-----|------|
| `Actor` (TileEntity 상속) | `ACardGameActor` (AGridEntity 상속) | Tick 기반 이동, AnimMontage 콜백 |
| `TurnManager` (MonoSingleton) | `UTurnSubsystem` (UWorldSubsystem) | 레벨 전환 시 자동 생성/파괴 |
| `StageManager` (MonoSingleton) | `UStageSubsystem` (UWorldSubsystem) | 레벨 전환 시 자동 생성/파괴 |
| `IEnumerator` 코루틴 | 콜백 체인 + FTimerHandle + Tick | 패턴별 최적 방식 선택 |
| `Animator.Play("Attack01")` | `UAnimMontage` + `OnMontageEnded` | AnimBP에서 설정 |
| `MoveTowards` (프레임 보간) | `VInterpConstantTo` (Tick) | SetActorTickEnabled로 On/Off |
| `Coroutine yield WaitForSeconds` | `GetTimerManager().SetTimer()` | 0.3초 피해 타이밍 등 |

### 코루틴 → UE5 변환 패턴

| Unity 패턴 | UE5 대체 | 용도 |
|-----------|---------|------|
| MoveCoroutine (프레임 보간) | Tick + VInterpConstantTo | 웨이포인트 이동 |
| AttackCoroutine (애니메이션 대기) | AnimMontage + OnMontageEnded | 공격/피격/회피/사망 |
| MonsterTurnCoroutine (순차 실행) | 콜백 체인 (ProcessNextMonster) | 몬스터 AI 순차 행동 |
| ApplyCardEffectsCoroutine (순차 효과) | 콜백 체인 (ProcessNextEffect) | 카드 효과 12종 |
| WaitForSeconds(0.3f) | SetTimer(0.3f) | 공격 중 피해 타이밍 |

### 순환 의존성 해결

- `UTurnSubsystem` ↔ `UStageSubsystem` 상호 참조
- **헤더**: forward-declare만 사용 (include 금지)
- **cpp**: `GetWorld()->GetSubsystem<상대>()` 패턴으로 런타임 접근

### 몬스터 AI (콜백 체인)

| 상태 | 행동 | 구현 |
|------|------|------|
| Alert (경계) | 최근접 아군 향해 이동 → 사거리 내면 공격 | GetNearestActor + GetBestTileToward + ExecuteAttack |
| Wander (배회) | 랜덤 2칸 이내 이동 | GetReachableTiles + RandRange |
| Alert 판정 | 임의 몬스터 Sight 범위 내 아군 존재 | CheckAlertState (Manhattan 거리) |

### 공격 실행 (ExecuteAttack)

```
1. Attacker->PerformAttack(Target, CheckBothDone)  // 공격 애니메이션
2. SetTimer(0.3초):
   - TakeDamage(AttackPower)                        // 피해 적용
   - Target->ReceiveHit(Attacker, CheckBothDone)    // 피격 애니메이션
3. 두 애니메이션 완료 (MakeShared<int32> 카운터):
   - 사망 시: Die → CheckBattleResult → OnComplete
   - 생존 시: OnComplete
```

### 카드 효과 12종 (ProcessNextEffect)

| 효과 | 처리 | 비동기 여부 |
|------|------|-----------|
| Damage | 명중 판정 → Attack→Hit→사망체크 | 비동기 (애니메이션 콜백) |
| Heal | DPawn/DMonster.Heal(Value) | 동기 |
| Shield | AddShield(Value) | 동기 |
| RestoreAction | TurnSubsystem.AddSharedActingPower | 동기 |
| Reload | DPawn.RestoreAmmo() | 동기 |
| DrawCard | DPawn.DrawCards(Value) | 동기 |
| BuffAttack/Armor/Movement | ApplyBuff(+Value, Duration) | 동기 |
| DebuffAttack/Armor/Movement | ApplyBuff(-Value, Duration), 미스 시 스킵 | 동기 |
