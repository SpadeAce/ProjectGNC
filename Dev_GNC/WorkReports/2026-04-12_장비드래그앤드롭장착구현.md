# 장비 드래그앤드롭 장착 기능 구현

- **일시**: 2026-04-12
- **분류**: 기능 포팅 / 리팩토링 (UI / 장비 시스템)
- **범위**: EquipDragDropOp.h(신규), EquipListItemWidget.h/.cpp, PawnManagePageWidget.h/.cpp

---

## 배경

Unity 원본에서는 미장착 장비를 드래그하여 슬롯에 드롭하는 방식으로 장착이 이루어졌다. UE5 포팅 시 이것이 좌클릭 선택 → 슬롯 클릭의 2단계 흐름으로 잘못 구현되어 있었다. 원본 방식대로 드래그앤드롭으로 수정하고, 드래그 중 마우스에 장비 위젯이 붙어 따라다니도록 구현했다.

## Unity 원본 동작

- `PawnManageEquipItem`: `IPointerDownHandler`/`IDragHandler`/`IPointerUpHandler` 구현
  - 드래그 시작 시 `transform.SetParent(_canvas.transform)` → Canvas 위로 올려 마우스를 따라다니게 처리
  - 드래그 종료 시 원래 위치/부모로 복귀
- `PawnManagePage.OnEquipItemDragEnd`: `RectTransformUtility.RectangleContainsScreenPoint`로 각 슬롯 영역 판정 → 타입 매칭 시 `EquipToSlot()` 호출

## 구현 내용

### 1. UEquipDragDropOp (신규)

기존 `UCardDragDropOp` 패턴을 따라 장비 전용 드래그 오퍼레이션 클래스 생성.

```cpp
UCLASS()
class UEquipDragDropOp : public UDragDropOperation
{
    UPROPERTY()
    TObjectPtr<UDEquipment> Equipment;
};
```

### 2. EquipListItemWidget — 드래그 시작

- `NativeOnMouseButtonDown`: 좌클릭 시 `UWidgetBlueprintLibrary::DetectDragIfPressed()` 호출로 드래그 감지 시작
- `NativeOnDragDetected`: `UEquipDragDropOp` 생성, `DefaultDragVisual = this` 설정 → 드래그 중 장비 위젯이 마우스를 따라다님 (`Pivot = CenterCenter`)
- 기존 `ButtonSelect` 클릭 바인딩, `FOnEquipItemClicked` 델리게이트, `NativeConstruct`/`NativeDestruct` 제거

### 3. PawnManagePageWidget — 드롭 수신

- `NativeOnDrop` 오버라이드 추가:
  1. `UEquipDragDropOp`으로 캐스트하여 페이로드 확인
  2. 드롭 좌표(`GetScreenSpacePosition`)를 각 슬롯 버튼의 `GetCachedGeometry()`와 비교하여 어느 슬롯에 떨어졌는지 판정
  3. 슬롯 타입 매칭 확인 → 기존 장비 있으면 해제 → 새 장비 장착 → UI 갱신

### 4. 2클릭 흐름 제거

- `HandleUnequippedItemClicked`, `HandleEquipItemSelected` 삭제
- `SelectedEquip` 멤버 제거
- `TileUnequippedList->OnItemClicked()` 바인딩 제거
- `HandleEquipSlotClicked` 간소화: 슬롯 클릭 시 장착 해제만 수행 (장착 로직 제거)

## 장착 흐름 비교

| 동작 | 이전 (2클릭) | 변경 후 |
|------|-------------|---------|
| 장비 장착 | 좌클릭 선택 → 슬롯 클릭 | 드래그 → 슬롯에 드롭 |
| 빠른 장착 | — | 우클릭 (자동 슬롯 배정) |
| 장비 해제 | 슬롯 클릭 | 슬롯 클릭 (동일) |

## 설계 포인트

- **기존 카드 드래그 패턴 재사용**: `UCardDragDropOp` + `BattleCardWidget`의 `DetectDragIfPressed` → `NativeOnDragDetected` → `NativeOnDrop` 패턴을 그대로 적용
- **DefaultDragVisual**: `this`를 설정하여 UE5 내장 드래그 비주얼 시스템이 위젯을 마우스에 붙여서 이동시킴. Unity의 수동 `transform.position = e.position` 구현 대비 코드 간소화
- **드롭 슬롯 판정**: `GetCachedGeometry().AbsoluteToLocal()`로 스크린 좌표를 슬롯 로컬 좌표로 변환 후 바운드 체크. Unity의 `RectangleContainsScreenPoint` 대응
