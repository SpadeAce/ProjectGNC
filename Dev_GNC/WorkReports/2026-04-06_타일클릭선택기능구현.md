# 마우스 클릭으로 타일 선택 기능 구현

- **일시**: 2026-04-06
- **분류**: 기능 구현 (입력 처리)
- **범위**: CardGamePlayerController.h/.cpp

---

## 배경

Unity 원본 `StageScript.Update()`에서 마우스 좌클릭 → Raycast → `StageManager.SelectTile()` 호출하는 입력 브릿지 로직이 UE5에 포팅되지 않았다. `StageSubsystem::SelectTile()` 선택 로직과 `SquareTile` 오버레이 시각 피드백은 이미 구현 완료 상태였으나, 실제 마우스 클릭을 감지하여 연결하는 코드가 누락되어 타일 선택이 동작하지 않았다.

---

## 변경 내용

### 1. CardGamePlayerController.h — 입력 프로퍼티 및 핸들러 추가

```cpp
// Enhanced Input 에셋 참조
UPROPERTY(EditDefaultsOnly, Category = "CardGame|Input")
TObjectPtr<UInputMappingContext> ClickMappingContext;

UPROPERTY(EditDefaultsOnly, Category = "CardGame|Input")
TObjectPtr<UInputAction> ClickAction;

// 오버라이드 및 핸들러
virtual void SetupInputComponent() override;
void HandleClick(const FInputActionValue& Value);
```

### 2. CardGamePlayerController.cpp — 클릭 입력 처리 구현

- **BeginPlay()**: `ClickMappingContext`를 Enhanced Input Subsystem에 등록 (priority 1)
- **SetupInputComponent()**: `ClickAction`을 `HandleClick`에 바인딩
- **HandleClick()**: 
  1. `StageSubsystem->bIsBusy` 체크
  2. `GetHitResultUnderCursorByChannel(ECC_Visibility)` Raycast
  3. Hit Actor를 `ASquareTile`로 캐스트
  4. `StageSubsystem->SelectTile()` 호출

## Unity 원본 참조

`Assets/Scripts/Logic/Scene/StageScript.cs` 64-93행:
```csharp
// Update() 에서 좌클릭 감지
if (Mouse.current.leftButton.wasReleasedThisFrame)
{
    if (StageManager.Instance.IsBusy) return;
    if (_stagePage != null && _stagePage.IsCardDragging) return;
    if (EventSystem.current != null && EventSystem.current.IsPointerOverGameObject()) return;
    HandleTileClick(Mouse.current.position.ReadValue());
}
```

UE5에서는 DragDrop 시스템이 드래그 중 클릭 이벤트를 소비하므로 `IsCardDragging` 별도 체크 불필요.

## 에디터 설정 필요 사항

1. **IA_TileClick** InputAction 에셋 생성 (ValueType: bool)
2. **IMC_TileClick** InputMappingContext 에셋 생성 → LMB Released에 IA_TileClick 매핑
3. PlayerController BP의 `ClickAction`, `ClickMappingContext`에 할당

## 검증 방법

- 타일 좌클릭 시 선택 오버레이 표시 확인
- 같은 타일 재클릭 시 선택 해제 확인
- 액터 있는 타일 선택 시 이동 범위 하이라이트 확인
- 카드 드래그 드롭 중 타일 클릭이 발생하지 않는지 확인
