# classref 타입 지원 및 add-table asset 매핑 개선

- **일시**: 2026-03-29
- **분류**: 기능 추가 (데이터 파이프라인 + 코드 생성)
- **범위**: Tools 파이프라인, add-table 스킬, CardGameRowTypes.h, DPawn.cpp, 가이드 문서

---

## 배경

스키마에서 `asset:CardGameActor`를 지정해도 에디터에서 BP 클래스 선택 드롭다운이 아닌 텍스트 입력창으로 표시되는 문제가 있었다. 원인은 두 가지:

1. **add-table 스킬이 proto 파일만 읽음** — `asset:X`가 proto에서 `string`으로 변환되므로, C++ 코드 생성 시 `FString`으로 매핑됨
2. **데이터 에셋 참조와 BP 클래스 참조의 구분 없음** — `TSoftObjectPtr`(에셋 인스턴스)과 `TSoftClassPtr`(BP 클래스)은 용도가 다르지만 구분할 수단이 없었음

---

## 변경 내용

### 1. 새 FieldType `classref:ClassName` 도입

| FieldType | 용도 | C++ 타입 |
|---|---|---|
| `asset:X` | 데이터 에셋 참조 | `TSoftObjectPtr<UX>` |
| `classref:X` | BP 클래스 참조 | `TSoftClassPtr<AX>` |

파이프라인 내부에서는 둘 다 `string`으로 처리. 차이는 C++ 코드 생성 시에만 발생.

### 2. 파이프라인 — `classref:X` 지원

| 파일 | 변경 |
|---|---|
| `Tools/validator.py` | `classref:` 시작 시 `asset:`와 동일하게 string 검증, 알 수 없는 타입 경고 제외 |
| `Tools/proto_generator.py` | `classref:X` → `"string"` 변환 추가 |
| `Tools/data_serializer.py` | `classref:` 시작 시 `str(value)` 반환 |
| `Tools/main.py` | asset 필드 보존 대상에 `classref:` 포함 |

### 3. add-table 스킬 — schema_snapshot 참조

`.claude/commands/add-table.md` 수정:
- 2단계(Proto 메시지 분석)에서 `Tools/output/schema_snapshot.json`도 함께 읽도록 변경
- `schema_snapshot`의 `field_type`이 proto 타입보다 우선 (원본 `asset:`/`classref:` 정보 보존)
- 타입 매핑 추가: `asset:X` → `TSoftObjectPtr<UX>`, `classref:X` → `TSoftClassPtr<AX>`
- forward declaration 자동 추가 규칙 명시
- `Variant_CardGame` → `CardGame` 경로 수정

### 4. 기존 코드 업데이트 — PawnDataRow

| 파일 | 변경 |
|---|---|
| `CardGameRowTypes.h` | `FPawnDataRow.PrefabPath`: `FString` → `TSoftClassPtr<ACardGameActor>`, forward declaration 추가 |
| `DPawn.cpp` | `IconPath = Data->PrefabPath` → `IconPath = Data->PrefabPath.ToString()` (2곳) |

### 5. 가이드 문서

`WorkReports/Guide/DataTable_가이드.md`:
