# Phase 2: 타일맵 + 그리드 시스템 구현

> **날짜**: 2026-03-23
> **작업 유형**: 기능 구현 (Unity→UE5 포팅)
> **Phase**: Phase 2 — 타일맵 + 그리드

---

## 작업 요약

Unity C# 카드게임의 타일맵/그리드 시스템을 UE5 C++로 포팅.
TileManager, SquareTile, TileSlot, TileEntity, EdgeTile, TileWall, TileMapPreset, EntityCatalog 전체를 UE5 아키텍처로 변환하였다.

## 주요 변환 사항

### 아키텍처 매핑

| Unity | UE5 | 비고 |
|-------|-----|------|
| `TileManager` (MonoSingleton) | `UGridSubsystem` (UWorldSubsystem) | 레벨 전환 시 자동 생성/파괴 |
| `SquareTile` (MonoBehaviour) | `ASquareTile` (AActor) | |
| `TileSlot` (MonoBehaviour) | `UGridSlotComponent` (UActorComponent) | ASquareTile/AEdgeTile에 부착 |
| `TileEntity` (abstract MonoBehaviour) | `AGridEntity` (Abstract AActor) | |
| `EdgeTile` (MonoBehaviour) | `AEdgeTile` (AActor) | |
| `TileMapPreset` (ScriptableObject) | `UTileMapPreset` (UDataAsset) | |
| `EntityCatalog` (ScriptableObject) | `UEntityCatalog` (UDataAsset) | |
| `GameObject prefab` | `TSoftClassPtr<AActor>` | 비동기 에셋 참조 |
| `System.Type` (리플렉션) | `UClass*` + `IsA()` | 우군 판별 |

### 좌표계 변환
- Unity: XZ 평면 (Y=Up), `Vector2Int.y` → Z축
- UE5: XY 평면 (Z=Up), `FIntPoint.Y` → Y축
- `GridToWorld`: `FVector(X * TileSize, Y * TileSize, 0.0f)`

### 경로탐색 알고리즘 (Unity 직역)
| 알고리즘 | 메서드 | 용도 |
|---------|--------|------|
| BFS | `GetReachableTiles()` | 이동 범위 내 도달 가능 타일 |
| A* | `GetPath()` | Manhattan 휴리스틱 최단 경로 |
| Greedy + Bresenham | `SmoothPath()` + `HasLineOfSight()` | 경로 스무딩, 직선 통과 검사 |

### 빌드 중 수정 사항
- `OnClicked` → `OnTileClicked`으로 변경 (AActor에 이미 `OnClicked` 존재하여 이름 충돌)
- `GridSubsystem.cpp`에 `DObject.h` include 추가 (`UDObject` 전방선언만으로는 `IsA()` 호출 불가)
