# TileMap 에디터 사용 가이드

> **목적**: UE5 에디터 내에서 타일맵 프리셋(`UTileMapPreset` 데이터 에셋)을 비주얼 그리드 기반으로 생성·편집하는 방법을 안내.

---

## 1. 에디터 열기

**도구 > CardGame > TileMap Editor** 메뉴를 클릭하면 도킹 가능한 탭이 열린다.
탭은 에디터 레이아웃 내 원하는 위치에 도킹하거나 플로팅 윈도우로 사용 가능.

---

## 2. 에디터 구성

에디터는 5개 영역으로 구성된다:

```
┌─ 설정 패널 ────────────────────────────────┐
│ [폭] [높이] [타일크기] [턴제한] [최대폰수] [생성] │
├─ 편집 모드 툴바 ───────────────────────────┤
│ [타일 타입] [스폰 포인트] [엔티티 배치]         │
├─ 도구 패널 ────────────────────────────────┤
│ 모드별 선택 컨트롤 + [전체 채우기] [전체 지우기] │
├─ 그리드 캔버스 (스크롤 가능) ──────────────┤
│ 색상 코딩된 타일 그리드 + 엣지 슬롯            │
├─ 저장/불러오기 패널 ───────────────────────┤
│ [내보내기] [불러오기] [저장(덮어쓰기)]          │
└────────────────────────────────────────────┘
```

---

## 3. 그리드 생성

1. 설정 패널에서 **폭**, **높이** 입력 (기본값: 5×5)
2. **타일크기**, **턴제한**, **최대폰수** 설정 (선택)
3. **[생성]** 버튼 클릭 → 모든 타일이 Ground 상태로 초기화된 그리드가 표시됨

> ⚠ 생성 시 기존 그리드 데이터는 초기화된다.

---

## 4. 편집 모드

### 4-1. 타일 타입 모드

타일의 지형 유형을 페인팅한다.

| 타입 | 색상 | 설명 |
|------|------|------|
| Empty | 회색 | 빈 공간 (타일 없음) |
| Ground | 초록 | 일반 이동 가능 지형 |
| Dirt | 갈색 | 이동 가능 지형 변형 |
| Water | 파랑 | 이동 가능 지형 변형 |
| Blocked | 빨강 | 이동 불가 지형 |

**조작법:**
- 도구 패널에서 타입 버튼 선택 → 그리드 셀 **클릭** 또는 **드래그**로 페인팅
- 선택된 타입 버튼은 해당 타일 색상으로 강조 표시

### 4-2. 스폰 포인트 모드

플레이어/적 유닛의 스폰 위치를 지정한다.

**조작법:**
- **Player** 또는 **Enemy** 버튼 선택 + **SpawnId** 입력
- **클릭**: 빈 셀에 스폰 포인트 배치 / 이미 있으면 토글(제거)
- **드래그**: 선택된 타입+ID로 연속 페인팅

**표시:**
- 플레이어 스폰: 셀 우상단에 시안색 `P:번호`
- 적 스폰: 셀 우상단에 마젠타색 `E:번호`

### 4-3. 엔티티 배치 모드

EntityCatalog에 등록된 엔티티를 타일 셀 또는 엣지 슬롯에 배치한다.

**조작법:**
- **Entity Id** 직접 입력하거나 **[카탈로그에서 선택...]** 버튼으로 팝업 메뉴에서 선택
- 그리드 셀 또는 엣지 슬롯(타일 사이 어두운 스트립) 클릭 → 컨텍스트 메뉴에서 엔티티 선택
- "None" 선택 시 엔티티 제거

**표시:**
- 타일 셀: 하단 중앙에 흰색 엔티티 이름 (최대 6자)
- 엣지 슬롯: 비어있으면 어두운 회색, 엔티티 있으면 주황색 + 이름 표시

**카탈로그 관리:**
- **[카탈로그 새로고침]** 버튼으로 콘텐츠 브라우저의 `UEntityCatalog` 데이터 에셋을 재검색
- 현재 로드된 카탈로그의 엔티티 수가 도구 패널에 표시됨

---

## 5. 유틸리티 버튼

| 버튼 | 동작 |
|------|------|
| **전체 채우기** | 현재 편집 모드의 선택값으로 모든 셀을 채움 |
| **전체 지우기** | 모든 타일을 Ground로 초기화, 스폰/엔티티/엣지 전부 제거 |

---

## 6. 저장 및 불러오기

### 6-1. 내보내기 (DataAsset)
1. **[내보내기 (DataAsset)]** 클릭
2. 콘텐츠 브라우저 저장 다이얼로그에서 경로와 이름 지정
   - 권장 경로: `/CardGame/DataAssets/`
   - 이름 예시: `TMP_Stage01`
3. `UTileMapPreset` 데이터 에셋이 생성되어 저장됨

### 6-2. 불러오기
1. **[불러오기]** 클릭
2. 에셋 피커에서 기존 `UTileMapPreset` 데이터 에셋 선택
3. 에디터에 프리셋 데이터가 로드되어 그리드에 표시됨

### 6-3. 저장 (덮어쓰기)
- 내보내기 또는 불러오기로 프리셋이 연결된 상태에서 **[저장 (덮어쓰기)]** 클릭
- 현재 그리드 상태가 기존 에셋에 즉시 반영됨

---

## 7. 그리드 읽는 법

### 타일 셀 (32×32 사각형)
- **배경 색상**: 타일 타입에 따라 결정 (4절 참조)
- **우상단 라벨**: 스폰 포인트 정보 (시안=Player, 마젠타=Enemy)
- **하단 라벨**: 배치된 엔티티 이름

### 엣지 슬롯 (타일 사이 좁은 스트립)
- 열 사이: 세로 12×32 스트립 (East 엣지)
- 행 사이: 가로 32×12 스트립 (North 엣지)
- 어두운 회색 = 비어있음, 주황색 = 엔티티 배치됨

### 좌표 체계
- 데이터상 Y=0이 하단, 화면상 Y=0이 상단 (Y축 반전 표시)
- 좌표는 `DT_StagePresetData` 및 `GridSubsystem::GenerateTileMapFromPreset()`과 동일 체계

---

## 8. 워크플로우 예시

### 새 스테이지 만들기
1. 에디터 열기 → 폭 8, 높이 6 입력 → [생성]
2. 타일 타입 모드에서 지형 페인팅 (Blocked로 벽 구역 표시)
3. 스폰 포인트 모드에서 Player(P:1~4), Enemy(E:1~3) 배치
4. 엔티티 모드에서 엣지 슬롯에 벽 엔티티 배치
5. [내보내기] → `TMP_Stage01`로 저장
6. `DT_StagePresetData`에 생성된 에셋 경로 등록

### 기존 스테이지 수정
1. [불러오기] → 기존 `TMP_Stage01` 선택
2. 원하는 타일/스폰/엔티티 수정
3. [저장 (덮어쓰기)] → 즉시 반영
